# Lang Battle (랑그릿사 1 오마주 전투 — 격리 실험)

> **격리 실험 기능.** 기존 게임(Game/Combat/Battle)과 **완전히 분리**된 별도 전투 씬이다.
> 브랜치 `test/lang1battle`에서 랑그릿사 1(메가드라이브) 전투 로직·연출을 오마주해
> 처음부터 새로 구현했다. 기존 캐릭터/부대 데이터를 전혀 참조하지 않고 **더미 데이터**로
> 10:10 교전을 재생한다.
>
> 참고 자료: 북미판 팬 영문 ROM 리버스 엔지니어링 스펙(사용자 제공) + 실기 영상.
> 원본 스프라이트/팔레트/음악/텍스트 등 창작 에셋은 재배포하지 않고 **도형 플레이스홀더**로
> 대체한다. 사용하는 것은 게임 메커니즘·수치·공식(아이디어/사실)뿐이다.

## 진입

- 타이틀 메뉴 **[새 게임]** → `res://scenes/lang_battle/lang_battle.tscn` 로 바로 진입.
- 기존 **[시작]** 버튼(→ `game.tscn`)은 그대로 둔다.

## 핵심 설계: 계산과 연출의 분리 (스펙 §0)

원작처럼 **Resolver(순수 계산)** 와 **Presenter(연출 상태머신)** 를 분리한다.
덕분에 "애니 스킵"이 결과를 바꾸지 않는다.

| 레이어 | 파일 | 역할 |
|:--|:--|:--|
| RNG | `lang_rng.gd` | 원본 LCG 변형 재현(§2.5). 결정론적. |
| 데이터 | `lang_data.gd` + `data/*.csv` | 클래스 스탯 90종 + 병종 상성(§1.1, §2.3). |
| Resolver | `lang_resolver.gd` | 스탯 조립·상성·지휘보정·교전·명중(§2.1~§2.4). 순수 함수. |
| Presenter | `lang_battle.gd` | 상태머신으로 결과 재생(§3.2). |
| 전장 렌더 | `lang_battlefield.gd` | 픽셀아트 전장 + 병사 난투 + 타격 이펙트(§4, §5.1). |
| HUD | `lang_hud.gd` | 3분할 정보 패널(병력·AT/DF·명중률·VS). |

## RNG (§2.5)

32비트 LCG 변형. 상태 상위 16비트만 유효.
```
s = state; if s==0: s = 0x2A6D365A
s = (s * 41) & 0xFFFFFFFF
r = ((s & 0xFFFF) + (s >> 16)) & 0xFFFF
state = r << 16
return r
```
검증 수열(상태 0에서 `next()%100`): `[43,43,79,79,51,99,99,3,27,63,3,51]` — 테스트로 대조.

## Resolver (§2.1~§2.4)

- **스탯 조립**: `at = base.at + typeBonus.atk + cmdBonus.at`, df도 동일.
- **병종 상성(§2.3)**: 상대의 `magicTier`(0~4)로 자기 상성표 조회. `matchup.csv` 사용.
- **지휘보정(§2.4)**: 맨해튼 거리 ≤ 지휘범위면 지휘관 AT/DF 보너스. 본인은 거리 0 → 항상 적용.
- **교전(§2.2)**: 양방향 실행. 공격자 `attack_count`회 시도 → 방어자 반격(`attack_count>0`일 때).
  - `attempt_hit`: 회피(명중기본 30 + 방어측 회피) → 특수 게이트(`rng%36`) → 데미지.
  - 데미지: `factor = 병사수*3+50 (+rng+10)`, `raw = at*factor/100`. `raw ≥ 방어측 df` 면 1명 전사.
- 병사 1명 = **8 세부단위**(strength). 표시 병력 = strength/8.

### 미구현/근사 (사실대로 표기)
- `attack_count`: 원본 "유닛 수 +3"의 정확한 유도가 불확실 → **병사 수와 동일**으로 근사.
- 특수 게이트(`rng%36`) 효과: 원본 미확정(부록A) → **데미지 없음(SPECIAL)** 으로 처리.
- 전투 후 처리(regen·전직, §2.6): **미구현**.
- 마법 경로(§6): **미구현**(물리 전투만).
- 아이템 보정 테이블(§1.3): 훅(`item_id`)만 있고 실제 보정 테이블은 **미로드**.

## Presenter — 상태 전이 (§3.2 + 전진·난투 연출)

```
CHARGE (화면 밖 넓은 대형에서 즉시 돌격 + 하단 숫자 동시)
→ CLASH (킬 스케줄 재생 — 사망을 전 구간에 퍼뜨림, 재교전으로 끝까지 난전)
→ POST (결과 확정 safety) → RETREAT (전원 공방 마치고 마지막에 함께 복귀) → DONE
```
- **스폰은 화면 밖(진영 바깥)에서 좌우 간격을 넓게** 하고 바로 돌격한다. 각 병사의 복귀 위치(`home`)는
  화면 안 진형(진영 끝 3행 [3,3,4]) 자리로, 전투 후 각자 그 자리로 돌아온다.
  하단 숫자(AT/DF 지휘보정 틱업·병력·명중률)는 돌격과
  **동시에** 진행(두 레이어 병렬). 정보 표시를 위해 돌격을 지연시키지 않는다.
- 양쪽에서 **첫 충돌**(`any_engaged`)이 생기면 CLASH로 전환(상한 2.2s) — 전원 도착을 기다리지 않는다.
- **아무 키** = 스킵(결과 불변), **ESC** = 타이틀 복귀, DONE에서 키 = 재전투.

### 돌격 → 난전 연출 (랑그릿사 1 롬 디스어셈블 그대로)
**중요**: 원본 롬(메가드라이브)을 직접 디스어셈블해 확인한 결과, **원본에도 병사가 개별적으로
움직인다**. (이전 스펙 §5.1의 "개별 캐릭터 모션 없음"은 하단 스탯 패널만 본 오류였다.)
전투는 **두 레이어가 동시에** 돈다: ① 하단 스탯 패널 상태머신(위 흐름) ② **맵 위 병사 스프라이트
애니메이션**. 아래는 ②의 롬 분석 결과이며 Presenter 레이어에서만 재현한다(Resolver·테스트 무관).

롬에서 파악한 사실(원본 루틴 주소 병기, 검증용):

| 항목 | 원본 값 | 루틴 |
|:--|:--|:--|
| 병사 스폰 (병종별 애니 템플릿) | fighter=`0x156E`, mage=`0x1CFC`, 기타=`0x1DAA` | 테이블 `0xFCBE` |
| 진영 기준 X | **좌 56 / 우 264** (center 160, `160±104`) | `0x156E`/`0xF4C2` |
| **돌격 속도** | **X = 3.0 / Y = 2.0 px/frame** (16.16 고정소수 `0x30000`/`0x20000`) | `0x156E` |
| **적 태그로 등속 접근** | X를 먼저 좁히고 **≤36px(`0x240000`)** 되면 Y 접근 | `0xE30E` |
| 대형 좌표로 이동 | 고정 좌표(`$6/$8`)로 등속 | `0xE37E` |
| **타겟 선택** | ① 날 노리는 적 > ② **미교전 적** > ③ 교전 중 적, 동순위 최근접(맨해튼) → **1:1 분산** | `0xE5DA` |
| **충돌 박스** | 병사마다 16px 박스(`$24=-8/$26=+8`) 겹침 판정 → 서로 밀림(부피) | `0xE87A` |
| 화면 클램프 | X∈[16,304] 등 플레이필드 안으로 | `0xE48E` |
| 걷기 프레임 | 이동량에 따라 카운터 증가 | `0x16E4` |
| 공격 | 효과음(Z80 `$A11100`, cmd `$32`) + **전방으로 날아가는 슬래시 이펙트**(3px/frame) | `0x1782`/`0x1860` |
| **사망** | **위로+뒤로 포물선 넉백 점프**(Y launch −2.5~−3.5px/f, X 뒤로 0.5~1.5px/f, 중력 +0.125/f²) → 착지하면 시체 눕기+**점멸** 후 제거 (現 구현: 넉백 유지, 비행=Hurt·착지=Death 주저앉기·페이드) | `0x1976`/`0x19EA`→`0x1B24` |
| 방향 | 진영 비트로 좌우 반전 | `$22` bit6 |

구현(`lang_battlefield.gd`) — 위 값을 384폭(×1.2)으로 환산해 재현:
- 병사 수 = 병력(`soldierCount` = 큰 숫자, 최대 = 병종 `maxSoldiers`). 병사 1명 = 스프라이트 1개.
- **기본 진형 = 위→아래 3행 [3,3,4] 쐐기꼴**(가운데 행만 중앙 쪽으로 튀어나옴). 진영 끝(가장자리)에 앵커, 중앙 방향으로 균일한 열.
  이 진형 좌표(`_formation_slots`)가 각 병사의 **복귀 위치(`home`)** — 전투 후 각자 이 자리로 그대로
  돌아오고, 죽은 병사의 자리는 비워진다.
- **스폰은 화면 밖(진영 바깥)에서 — 좌우(열) 간격을 진형의 3배로 넓게**(`_spawn_slots`), 유닛별로 X를
  좌우 지터(±유닛 너비 ×2.5 ≈ 좌우 합 500%)로 흩뿌린다. 여기서 돌격해
  들어온 뒤, 전투가 끝나면 화면 안 진형 자리(`home`)로 복귀한다. 스폰과 복귀는 같은 (행,열) 순서라
  병사별로 1:1 대응된다.
- **돌격 속도는 전원 동일**(원본 등속 3/2px). 속도 지터는 두지 않는다.
- **스폰 즉시 돌격**: 각 병사가 적을 향해 **등속 접근**(`_move_step`, 원본 `0xE30E`): X 먼저(≤43px)→Y.
- **타겟팅**(`_retarget_all`, 원본 `0xE5DA` 재현): 매 프레임 **상대 적의 타겟 상태로 3단계 우선순위**
  — ① 그 적이 나를 노림(상호 락) > ② 미교전 적 > ③ 딴 아군과 교전 중인 적 — 동순위는 **맨해튼 최근접**
  (동거리는 인덱스 낮은 적). 우선순위는 **직전 프레임의 target 스냅샷**으로 매겨 양쪽을 일괄 커밋한다(프레임 순서 무관).
  코드에 있는 규칙은 이 3단계+최근접뿐이다. 여기서 나오는 **거동**으로 상호 락(①)+교전 회피(③ 강등)에 의한
  **1:1 분산**(2병사 기준 테스트로 검증)과, 좌우로 마주 선 진형에서 `|ΔX|`가 사실상 상수여서 **같은 행 적이 최근접**이
  되는 "정면 적 최우선"이 따라온다. 원본엔 "같은 열의 적을 노려라"는 전용 규칙이 없고, 다수 병사의 3열 레인 대치는
  이 규칙 + X우선 접근(`0xE30E`) + 분리(`0xE87A`)가 함께 만드는 창발 거동이다(런타임 관찰).
- **접전 간격**: 마주 선 적과 `GAP_X`(21px) 유지(`_move_step`) — 스프라이트 몸통(~16px) 감안해 살짝 띄움(겹쳐 싸우는 인상 완화).
- **분리**(`_separate`, 원본 충돌 박스 `0xE87A` 근사): `SEP_DIST`(15px) 안 겹치면 밀어내 **난전에 부피**(몸통 ~16px 맞춰 밀집 완화).
  **delta 기반**으로 부드럽게 수렴(프레임레이트 무관)하고, **접전(MELEE) 병사는 분리에서 제외**해 **제자리에서 교전**한다
  (밀림·미끄러짐 최소화). MELEE 병사는 obstacle(부피)로는 남아 CHARGE 병사가 피해 흐른다. 접전 중 지터 진폭도 축소.
- **첫 충돌 즉시 교전**(`any_engaged`) — 전원 도착을 기다리지 않는다. 사망/타격은 접전 중(MELEE) 우선.
- **자동 공방 cadence**(`_auto_strike`): 접전(MELEE) 병사는 `STRIKE_INTERVAL`(0.34s±지터)마다 자기 타겟을 향해
  주기적으로 찌른다(섬광+약한 흔들림). 병사가 **전투 내내, 그리고 복귀 대기 중에도 계속 싸우게** 해 idle 정지를 없앤다.
  타겟이 죽으면 다음 프레임 재타겟(재교전, `_retarget_all`). 살상(킬)은 Presenter가 결과대로 별도 지정.
- **펜싱 밀림**(`_auto_strike`): 공방 1회마다 **찌르는 쪽 전진 + 맞는 쪽 뒤로**를 같은 벡터(`face × FENCE_PUSH`=8px)로
  **`push_rem`에 예약**하고, `_process`가 매 프레임 이징(`PUSH_SMOOTH`)으로 소진 → **순간이동 아니라 슬라이드 애니메이션**.
  같은 벡터라 **거리 유지한 채 쌍이 좌우로 실제 이동**(`pos` 변경). 마주 선 두 병사가 번갈아 찔러 좌우로 티격태격
  밀리는 펜싱 인상. MELEE는 분리 제외라 이 밀림이 난전 내 주 위치 변화.
  (거리 유지는 근사 — 필드 가장자리 클램프 `[16,368]` 시 한쪽만 멈춰 거리가 잠깐 어긋날 수 있음. 중앙 난전이라 드묾.)
  (죽음 직전 밀림 시퀀스·공방 횟수(1~3회)는 **미구현/보류** — 이번엔 상시 밀림만.)
- **CLASH — 킬 스케줄**(`_build_plan`): **킬(사망) 이벤트만** 만들어(=시작−생존, 양쪽 번갈아) 재생. 총합은
  **Resolver 결과 그대로**(사망 수 불변, 테스트로 검증).
- **최소 전투시간**(`_melee_duration`): CLASH 길이를 **킬 수에서 분리**해 아래 기준으로 잡는다 — 적게 죽어도 너무
  빨리 끝나지 않게. CLASH는 이 시간 동안 지속하고(고정 간격 아님), 킬은 그 안에 고르게 분산, **남는 시간은 자동
  공방(스커미시)이 채운다**.
  ```
  deaths_a = a_start − a_final,  deaths_b = b_start − b_final
  basis    = max( min(a_start, b_start), deaths_a, deaths_b )
  melee_dur = clamp(MELEE_PER_UNIT × basis, MELEE_FLOOR, MELEE_CAP)   # 0.22 / 0.5 / 2.2(초)
  ```
  킬 재생 시각(`_schedule_times`): 등간격(`melee_dur/킬수`) 기준에서 **±`KILL_JITTER`(0.65)×간격 지터 후 정렬** —
  버스트·소강 리듬을 만들어 기계적 등간격을 깬다(킬 수·총 길이 불변, 연출 전용 RNG).
  - **min(a,b)**(동시 듀얼 수)가 바닥. **소수가 다수를 잡으면**(예: 1이 5킬) `max(deaths)`가 시간을 끌어올림.
    → 10:10 최장(2.2s cap), 1:10 즉패면 0.5s(floor), 1이 전멸시키면 2.2s.
  - CLASH 종료: **킬 소진 && `_timer ≥ melee_dur` && 진행 중 듀얼 없음**(`duels_active`) — 마지막 듀얼이
    사망까지 끝난 뒤 복귀 시작(복귀 중 사망 방지). 남는 시간은 스커미시가 채움.
  - **kill(side) → 듀얼 밀당 → 사망**: 접전(MELEE 우선) 병사 1명을 **패자 V**로 고르고, **승자 W = V의 현재 타겟**(적).
    즉사 대신 **듀얼**을 연다 — 공방 `N = randi(1..3)`회, **N−1번 밀림 스텝**(간격 `DUEL_STEP`≈0.18s) 뒤 V 사망(`_die`:
    **넉백(Hurt 비행) → 착지 후 주저앉기(Death) → 페이드**, 섬광+흔들림 — §"병사 스프라이트" 사망 연출 참조).
    - 각 밀림 스텝 확률(`_push_kind_for`): **60% 무이동 / 30% 패자 밀림(승자 전진) / 10% 승자 밀림(패자 전진)** —
      **패자 편향이되 확정 아님**. 밀림은 (a)의 `push_rem` 재사용(거리 유지·이징 슬라이드, 밀리는 쪽이 상대 반대로 물러남).
    - **W 없으면(타겟 null/사망) 즉사 fallback**(밀림 0). `N=1`도 밀림 0 → 바로 사망.
    - V는 `DUEL` 상태: **재타겟 대상 제외·자동 공방 안 함**. HUD 병력 카운트는 **kill 발화 시 감소**(시각적 사망은 밀당 끝난 뒤).
      duel 진행 중이면 `all_returned` 미완료(복귀는 duel 끝까지 대기).
  - **재교전**: 킬로 타겟이 죽으면 다음 프레임 `_retarget_all`이 남은 적으로 자동 재교전(1:N 난전) — 전투 중 이탈은 없다.
  - **복귀는 마지막에 한 번**: 스케줄 소진 후 `begin_retreat`(`_retreating`) → 각 전투 병사가 **진행 중이던 공방을 마치면**
    (`_retreat_ready`: `strike_t<=0`) `home`으로 복귀(`_return_step`). 끝까지 싸운 뒤 다 함께 돌아가는 그림 — idle 대기 없음.
    완료(`all_returned` = 생존자 전원 IDLE) 또는 상한에 DONE.
  - **찌르기 런지 ①은 펜싱 밀림(`_auto_strike`)으로 대체 활성**. **창 뻗기 ②·전방 슬래시 이펙트 ③은 임시 비활성**
    (코드 `[임시 비활성]` 마커, 재활성 시 주석만 해제). 공격 모션은 스프라이트 `attack` 애니, 전사는 `hurt`(넉백 비행)→`death`(착지 주저앉기) 애니가 담당. 섬광/화면 흔들림 유지.
- 패한 쪽은 **그 자리에서 전멸**(도망/복귀 없음). 생존자만 필드에 남음.
- 스킵 시 `force_result`로 즉시 최종 생존자만 남긴다.
- 큰 프레임 델타(로딩·랙)가 상태를 건너뛰지 않도록 `_process` 델타를 0.05로 상한.

### 픽셀아트 렌더링 (16비트풍)
원본은 16비트 도트 그래픽이다. 이를 위해 `Battlefield` 노드를 384×216 논리 공간에서
**안티앨리어싱 없는 `draw_rect` 픽셀로만** 그린 뒤 씬에서 `scale = ×5` 로 확대(→1920×1080)한다.
draw_rect 는 하드엣지라 확대해도 도트가 선명하다(SubViewport 불필요).
- 지형: 녹색 이끼 석조 벽돌 바닥 + 상단 성벽/구덩이 + 나무 사다리(원본 요새 맵 오마주). `_draw` 로 그린다.
- 병사: **`AnimatedSprite2D` 스프라이트**로 그린다(§아래 "병사 스프라이트"). 그림자·타격 섬광만 `_draw`.
- 하단 HUD 는 가독성을 위해 풀해상도 CanvasLayer 로 별도(3분할 금테 패널·포트레이트).

#### 병사 스프라이트 (임시 경보병 애셋)
병사는 `_units`(y-sort Node2D) 아래 병사 id별 `AnimatedSprite2D` 풀로 그린다(`_sync_sprites`).
지형(`_draw`)은 항상 뒤, 스프라이트는 y-sort 로 뒤→앞 정렬. 사망 중(DYING)도 스프라이트로 그려진다.
- **애셋**: Tiny RPG Character Pack — 팀 0=Soldier, 팀 1=Orc. `assets/units/{soldier,orc}_{walk,idle,attack,hurt,death}.png`.
  프레임 100×100(여백 포함), 실제 몸통 ≈17×22px. **개발용 플레이스홀더**(재배포 지양).
- **스케일**: 몸통 ≈17px → 바닥 타일 1칸(16px)에 맞춰 `SPRITE_SCALE = 16/17` 축소. `texture_filter=NEAREST`.
- **애니**(시트 가로 나열 → `SpriteFrames` 코드 구성):
  - `walk`(8f/10fps·루프) — 이동(`CHARGE`/`RETURN`). 생성 시 프레임 위상 랜덤(발맞춤 방지).
  - `idle`(6f/6fps·루프) — 정지(`MELEE`/`IDLE`/`DUEL`, 공격 아님).
  - `attack`(6f/18fps·1회) — `_auto_strike` 때 `atk_fire` 신호로 0프레임부터 재생. 애셋이 제자리 전방 찌르기라 펜싱과 정합.
  - `hurt`(4f/12fps·1회) — 사망 넉백 **비행 중** 자세(뒤로 젖힘).
  - `death`(4f/10fps·1회) — 착지 후 **제자리 주저앉기**(눕기).
  - `flip_h = face<0`(애셋 기본 우향).
- **정렬**: 프레임 중심(centered)에 병사 `pos` → 발끝 ≈ `pos.y+8.5`(그림자 `pos.y+8`와 일치).
- **피격 흰색 플래시**: 병사별 `ShaderMaterial`(`hit_flash.gdshader`) — `COLOR.rgb = mix(COLOR.rgb, 흰색, flash·COLOR.a)`. 진입 시 `COLOR`는 이미 (텍스처×modulate) 합성값이라 텍스처를 재샘플·재곱하지 않는다(그러면 `tex²`로 어두워짐). `flash=0`이면 기본 렌더와 픽셀 동일, 알파·사망 페이드도 그대로 보존.
  맞는 순간 `hit_t = HIT_FLASH_DUR`(0.12s) → 매 프레임 `flash = hit_t/DUR`(1→0 감쇠)로 팝. 트리거: `_auto_strike`의 타겟, 듀얼 밀당 패자(매 스텝), `_die`(치명타).
- **사망 연출(넉백→쓰러짐 하이브리드)** — 원본 롬 포물선 넉백을 유지하되 애셋과 화해:
  1. `_die` → `airborne=true`, 위로+뒤로 포물선 launch(`DEATH_VY`/`DEATH_VX`/`DEATH_GRAV`). 비행 중 `hurt` 자세. 그림자는 지면(`land_y`)에 고정 → 점프처럼.
  2. 착지(`pos.y ≥ land_y`) → `airborne=false`, `death_t=0` → `death`(주저앉기) 재생(`DEATH_COLLAPSE`≈0.4s).
  3. 이후 `DEATH_FADE`(0.35s) 동안 alpha↓ → 제거(스프라이트 풀도 정리).

### 테스트 시나리오 — 최소 전투시간 (`_melee_duration`)
`melee_dur = clamp(0.22 × max(min(a_start,b_start), deaths_a, deaths_b), 0.5, 2.2)`
- [정상] 10:10, 저데미지(생존 8/8, deaths 2/2) → basis=max(10,2,2)=10 → **2.2s**(cap). 적게 죽어도 안 빨리 끝남.
- [정상] 10:10, 고데미지(생존 5/3, deaths 5/7) → basis=max(10,5,7)=10 → **2.2s**. 길이는 min이 지배.
- [정상] 4:5(생존 1/2, deaths 3/3) → basis=max(4,3,3)=4 → **0.88s**.
- [경계] 1:10, 1 즉패(생존 0/10, deaths 1/0) → basis=max(1,1,0)=1 → **0.5s**(floor). 대부분 케이스.
- [정상] 1:10, 1이 5킬 후 사망(생존 0/5, deaths 1/5) → basis=max(1,1,5)=5 → **1.1s**. 소수가 다수를 잡음.
- [경계] 1:10, 1이 전멸(생존 1/0, deaths 0/10) → basis=max(1,0,10)=10 → **2.2s**(cap). 서사적.
- [경계] 무사망(deaths 0/0) → basis=min(a,b) → floor~cap 사이(킬 0이어도 스커미시로 시간 채움).

### 테스트 시나리오 — 듀얼 밀당 (`_push_kind_for`, 임계 0.6/0.9)
- [정상] `r<0.6` → NONE(무이동, 60%): 0.0, 0.59 → PUSH_NONE
- [정상] `0.6≤r<0.9` → LOSER(패자 밀림, 30%): 0.6, 0.89 → PUSH_LOSER
- [정상] `0.9≤r` → WINNER(승자 밀림, 10%): 0.9, 0.99 → PUSH_WINNER
- [경계] 공방 횟수 `_duel_count`는 `[1, 3]` 범위(랜덤).
