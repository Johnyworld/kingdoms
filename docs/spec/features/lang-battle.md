# Lang Battle (랑그릿사 1 오마주 전투 — 격리 실험)

> **격리 실험 기능.** 기존 게임(Game/Combat/Battle)과 **완전히 분리**된 별도 전투 씬이다.
> 브랜치 `test/lang1battle`에서 랑그릿사 1(메가드라이브) 전투 로직·연출을 오마주해
> 처음부터 새로 구현했다. 기존 캐릭터/부대 데이터를 전혀 참조하지 않고 **더미 데이터**로
> 10:10 교전을 재생한다.
>
> 참고 자료: 북미판 팬 영문 ROM 리버스 엔지니어링 스펙(사용자 제공) + 실기 영상.
> 원본 스프라이트/팔레트/음악/텍스트 등 창작 에셋은 재배포하지 않고 **도형 플레이스홀더**로
> 대체한다. 사용하는 것은 게임 메커니즘·수치·공식(아이디어/사실)뿐이다.

## 진입

- 타이틀 메뉴 **[전투 테스트]**(우측 상단) → **전투 설정 화면** `res://scenes/lang_setup/lang_setup.tscn` → [전투 시작] → `res://scenes/lang_battle/lang_battle.tscn`.
- 기존 **[시작]** 버튼(→ `game.tscn`)은 그대로 둔다.
- 전투 씬에는 **선택 버튼(우측 상단 시나리오 버튼)이 없다** — 전투 구성은 전적으로 설정 화면에서 정한다.
- 전투 씬 입력: **아무 키 = 스킵**, **마우스 클릭 = 같은 구성으로 재전투**(`_restart`, 언제든), **ESC = 전투 설정 화면으로 복귀**, DONE에서 아무 키 = 재전투.

### 전투 설정 화면 (`scenes/lang_setup/`)

양 진영(아군=side0/청, 적군=side1/적)의 전투 구성을 고른다.

| 항목 | 선택지 | 초기값 | 범위 |
|:--|:--|:--|:--|
| 병종 | 영웅 / 경보병 / 경궁병 | 경보병 | 진영별 |
| 숫자 | 1~10 | 10 | 진영별 |
| 교전 | 근접 / 원거리 | 근접 | **양 진영 공용(가운데 1개)** |

- **원거리는 기본 비활성**. **두 진영 중 하나라도 경궁병**이면 켜지고, 경궁병이 하나도 없으면 다시 비활성 되며 **근접으로 되돌아간다**.
- [전투 시작] → 선택값을 `LangBattleConfig`(static, `lang_battle_config.gd`)에 `{a, b, mode}`로 담아 lang_battle 로 전달. ESC = 타이틀.
- **이전 선택 기억**: 마지막으로 고른 값을 `LangBattleConfig._last`(소비 안 함)에 저장 → 전투 씬에서 ESC로 돌아오면 설정 화면이 그 값으로 복원(`_restore_last`). 세션 최초 진입 시엔 초기값.

### 커스텀 전투 매핑 (`LangBattleConfig` → `_load_custom`)

**경궁병은 항상 사격**한다. 단 **경궁병 vs 근접유닛**에서는 mode(근접/원거리)가 상대의 행동을 가른다:

- **양측 경궁병** → `_load_custom_ranged`: 사격 대결(`resolve_ranged` 1볼리씩, `St.RANGED`, `setup_ranged`). 서로 활 쏘고 마무리. **mode 무관**(근접/원거리 동일).
- **경궁병 + 근접유닛(경보병/영웅)**:
  - **원거리** → `_load_custom_pure_ranged`: 궁병만 사격(1볼리), 상대는 **제자리에서 맞기만**(반격·돌격·근접 없음). 상대가 영웅이면 진형 중앙 1스프라이트로 서서 화살에 HP만 깎임(`setup_ranged` 영웅 분기, 착탄마다 −1). `_then_melee=false`.
  - **근접** → `_load_custom_skirmish`: 궁병 사격 오프닝(1볼리) → 상대가 접근하면 근접 전환. 궁병이 side0/side1 어느 쪽이든 대응(`_sk_archer_side`).
    - 돌격측이 **영웅**이면 `setup_skirmish(..., charger_kind="hero")`로 1스프라이트 영웅 돌격, 볼리는 영웅 HP를 깎되(살상 화살 착탄마다 −1, `_update_arrows` 영웅 분기·`doomed` 예약 생략) 즉사 안 함. `_begin_skirmish_melee`가 생존 HP로 영웅 근접 유닛 조립.
    - 돌격측이 **경보병**이면 사격 생존 인원으로 근접(기존 스커미시).
  - **영웅 화살 회피(`HERO_ARROW_EVASION=40`)**: 영웅이 사격 표적이면 순수 사격·스커미시 볼리 모두에 회피 40 부여(경보병 돌격 회피는 `SCENARIO2_CHARGE_EVASION=25` 유지). 근접 탱킹(DF/HP)이 사격엔 안 통해 영웅이 볼리에 녹던 문제 보정 — **사격 resolve에서만** 적용해 영웅 근접 밸런스(vs 보병 69%)는 불변. 결과: 스커미시 궁병10 vs 영웅10 ≈ 영웅 30%(호각), 순수 사격 영웅 ~8/10 생존.
- **경궁병 없음(경보병/영웅끼리)** → `_load_custom_melee`: `resolve_engagement` + CHARGE→CLASH→RETREAT + `setup_custom`. (이 조합은 원거리 선택 자체가 비활성 → 항상 근접.)
- 병종별 유닛: 영웅=지휘관 클래스(classId 4, 27/24, `self_cmd=false`, count=HP/몫, 1스프라이트), 경보병=classId 1·`kind="infantry"`, 경궁병=classId 1·`kind="archer"`(근접 시 병종 상성 페널티).
- 인원은 side별 임의(1~10). HUD 병력·킬 스케줄·최소 전투시간 모두 side별 시작 값 기준(영웅은 HP=병력, `alive_count`도 영웅은 HP 반환).
- 설정이 없으면(예: 타이틀 없이 직접 씬 실행) 기존대로 **시나리오 1**로 진입(`_load_scenario` 폴백 — 직접 진입 전용, 버튼 없음).

## 핵심 설계: 계산과 연출의 분리 (스펙 §0)

원작처럼 **Resolver(순수 계산)** 와 **Presenter(연출 상태머신)** 를 분리한다.
덕분에 "애니 스킵"이 결과를 바꾸지 않는다.

| 레이어 | 파일 | 역할 |
|:--|:--|:--|
| RNG | `lang_rng.gd` | 원본 LCG 변형 재현(§2.5). 결정론적. |
| 데이터 | `lang_data.gd` + `data/*.csv` | 클래스 스탯 90종 + 병종 상성(§1.1, §2.3). |
| Resolver | `lang_resolver.gd` | 스탯 조립·상성·지휘보정·교전·명중(§2.1~§2.4). 순수 함수. |
| Presenter | `lang_battle.gd` | 상태머신으로 결과 재생(§3.2). |
| 전장 렌더 | `lang_battlefield.gd` | 픽셀아트 전장 + 병사 난투 + 타격 이펙트(§4, §5.1). |
| HUD | `lang_hud.gd` | 3분할 정보 패널(병력·AT/DF·명중률·VS). |

## RNG (§2.5)

32비트 LCG 변형. 상태 상위 16비트만 유효.
```
s = state; if s==0: s = 0x2A6D365A
s = (s * 41) & 0xFFFFFFFF
r = ((s & 0xFFFF) + (s >> 16)) & 0xFFFF
state = r << 16
return r
```
검증 수열(상태 0에서 `next()%100`): `[43,43,79,79,51,99,99,3,27,63,3,51]` — 테스트로 대조.

## Resolver (§2.1~§2.4)

- **스탯 조립**: `at = base.at + typeBonus.atk + cmdBonus.at`, df도 동일.
- **병종 상성(§2.3)**: 상대의 `magicTier`(0~4)로 자기 상성표 조회. `matchup.csv` 사용.
- **지휘보정(§2.4)**: 맨해튼 거리 ≤ 지휘범위면 지휘관 AT/DF 보너스. 본인은 거리 0 → 항상 적용. **단 유닛 `self_cmd=false`면 자기 지휘보정을 받지 않는다**(단독 영웅 — 지휘할 부대가 없음, 시나리오 5).
- **교전(§2.2)**: 양방향 실행. 공격자 `attack_count`회 시도 → 방어자 반격(`attack_count>0`일 때).
  - **`attack_count = 병사수 + 3`**(원작 §1.2/§2.2). 소수 유닛도 최소 4회 공격 → **보병 1명이 궁병 여럿을 잡을 수 있음**. 한 교전 사망 ≤ 공격측 attack_count.
  - `attempt_hit`: 회피(명중기본 30 + 방어측 회피) → 특수 게이트(`rng%36`) → 데미지.
  - 데미지: `factor = 병사수*3+50 (+rng+10)`, `raw = at*factor/100`. `raw ≥ 방어측 df` 면 1명 전사.
- 병사 1명 = **8 세부단위**(strength). 표시 병력 = strength/8.

### 미구현/근사 (사실대로 표기)
- 특수 게이트(`rng%36`) 효과: 원본 미확정(부록A) → **데미지 없음(SPECIAL)** 으로 처리.
- 전투 후 처리(regen·전직, §2.6): **미구현**.
- 마법 경로(§6): **미구현**(물리 전투만).
- 아이템 보정 테이블(§1.3): 훅(`item_id`)만 있고 실제 보정 테이블은 **미로드**.

## Presenter — 상태 전이 (§3.2 + 전진·난투 연출)

```
CHARGE (화면 밖 넓은 대형에서 즉시 돌격 + 하단 숫자 동시)
→ CLASH (킬 스케줄 재생 — 사망을 전 구간에 퍼뜨림, 재교전으로 끝까지 난전)
→ POST (결과 확정 safety) → RETREAT (전원 공방 마치고 마지막에 함께 복귀) → DONE
```
- **스폰은 화면 밖(진영 바깥)에서 좌우 간격을 넓게** 하고 바로 돌격한다. 각 병사의 복귀 위치(`home`)는
  화면 안 진형(진영 끝 3행 [3,3,4]) 자리로, 전투 후 각자 그 자리로 돌아온다.
  하단 숫자(AT/DF 지휘보정 틱업·병력·명중률)는 돌격과
  **동시에** 진행(두 레이어 병렬). 정보 표시를 위해 돌격을 지연시키지 않는다.
- 양쪽에서 **첫 충돌**(`any_engaged`)이 생기면 CLASH로 전환(돌격 상한 `ADVANCE_TIME`) — 전원 도착을 기다리지 않는다.
- **아무 키** = 스킵(결과 불변), **ESC** = 타이틀 복귀, DONE에서 키 = 재전투.

### 돌격 → 난전 연출 (랑그릿사 1 롬 디스어셈블 그대로)
**중요**: 원본 롬(메가드라이브)을 직접 디스어셈블해 확인한 결과, **원본에도 병사가 개별적으로
움직인다**. (이전 스펙 §5.1의 "개별 캐릭터 모션 없음"은 하단 스탯 패널만 본 오류였다.)
전투는 **두 레이어가 동시에** 돈다: ① 하단 스탯 패널 상태머신(위 흐름) ② **맵 위 병사 스프라이트
애니메이션**. 아래는 ②의 롬 분석 결과이며 Presenter 레이어에서만 재현한다(Resolver·테스트 무관).

롬에서 파악한 사실(원본 루틴 주소 병기, 검증용):

| 항목 | 원본 값 | 루틴 |
|:--|:--|:--|
| 병사 스폰 (병종별 애니 템플릿) | fighter=`0x156E`, mage=`0x1CFC`, 기타=`0x1DAA` | 테이블 `0xFCBE` |
| 진영 기준 X | **좌 56 / 우 264** (center 160, `160±104`) | `0x156E`/`0xF4C2` |
| **돌격 속도** | **X = 3.0 / Y = 2.0 px/frame** (16.16 고정소수 `0x30000`/`0x20000`) | `0x156E` |
| **적 태그로 등속 접근** | X를 먼저 좁히고 **≤36px(`0x240000`)** 되면 Y 접근 | `0xE30E` |
| 대형 좌표로 이동 | 고정 좌표(`$6/$8`)로 등속 | `0xE37E` |
| **타겟 선택** | ① 날 노리는 적 > ② **미교전 적** > ③ 교전 중 적, 동순위 최근접(맨해튼) → **1:1 분산** | `0xE5DA` |
| **충돌 박스** | 병사마다 16px 박스(`$24=-8/$26=+8`) 겹침 판정 → 서로 밀림(부피) | `0xE87A` |
| 화면 클램프 | X∈[16,304] 등 플레이필드 안으로 | `0xE48E` |
| 걷기 프레임 | 이동량에 따라 카운터 증가 | `0x16E4` |
| 공격 | 효과음(Z80 `$A11100`, cmd `$32`) + **전방으로 날아가는 슬래시 이펙트**(3px/frame) | `0x1782`/`0x1860` |
| **사망** | **위로+뒤로 포물선 넉백 점프**(Y launch −2.5~−3.5px/f, X 뒤로 0.5~1.5px/f, 중력 +0.125/f²) → 착지하면 시체 눕기+**점멸** 후 제거 (現 구현: 넉백 유지, 비행=Hurt·착지=Death 주저앉기·페이드) | `0x1976`/`0x19EA`→`0x1B24` |
| 방향 | 진영 비트로 좌우 반전 | `$22` bit6 |

구현(`lang_battlefield.gd`) — 위 값을 384폭(×1.2)으로 환산해 재현:
- 병사 수 = 병력(`soldierCount` = 큰 숫자, 최대 = 병종 `maxSoldiers`). 병사 1명 = 스프라이트 1개.
- **기본 진형 = 위→아래 3행 [3,3,4] 쐐기꼴**(가운데 행만 중앙 쪽으로 튀어나옴). 진영 끝(가장자리)에 앵커, 중앙 방향으로 균일한 열.
  이 진형 좌표(`_formation_slots`)가 각 병사의 **복귀 위치(`home`)** — 전투 후 각자 이 자리로 그대로
  돌아오고, 죽은 병사의 자리는 비워진다.
- **스폰은 화면 밖(진영 바깥)에서 — 좌우(열) 간격을 진형의 3배로 넓게**(`_spawn_slots`), 유닛별로 X를
  좌우 지터(±유닛 너비 ×2.5 ≈ 좌우 합 500%)로 흩뿌린다. 여기서 돌격해
  들어온 뒤, 전투가 끝나면 화면 안 진형 자리(`home`)로 복귀한다. 스폰과 복귀는 같은 (행,열) 순서라
  병사별로 1:1 대응된다.
- **돌격 속도는 전원 동일**(원본 등속 3/2px). 속도 지터는 두지 않는다.
- **스폰 즉시 돌격**: 각 병사가 적을 향해 **등속 접근**(`_move_step`, 원본 `0xE30E`): X 먼저(≤43px)→Y.
- **타겟팅**(`_retarget_all`, 원본 `0xE5DA` 재현): 매 프레임 **상대 적의 타겟 상태로 3단계 우선순위**
  — ① 그 적이 나를 노림(상호 락) > ② 미교전 적 > ③ 딴 아군과 교전 중인 적 — 동순위는 **맨해튼 최근접**
  (동거리는 인덱스 낮은 적). 우선순위는 **직전 프레임의 target 스냅샷**으로 매겨 양쪽을 일괄 커밋한다(프레임 순서 무관).
  코드에 있는 규칙은 이 3단계+최근접뿐이다. 여기서 나오는 **거동**으로 상호 락(①)+교전 회피(③ 강등)에 의한
  **1:1 분산**(2병사 기준 테스트로 검증)과, 좌우로 마주 선 진형에서 `|ΔX|`가 사실상 상수여서 **같은 행 적이 최근접**이
  되는 "정면 적 최우선"이 따라온다. 원본엔 "같은 열의 적을 노려라"는 전용 규칙이 없고, 다수 병사의 3열 레인 대치는
  이 규칙 + X우선 접근(`0xE30E`) + 분리(`0xE87A`)가 함께 만드는 창발 거동이다(런타임 관찰).
- **접전 간격**: 마주 선 적과 `GAP_X`(21px) 유지(`_move_step`) — 스프라이트 몸통(~16px) 감안해 살짝 띄움(겹쳐 싸우는 인상 완화).
- **분리**(`_separate`, 원본 충돌 박스 `0xE87A` 근사): `SEP_DIST`(15px) 안 겹치면 밀어내 **난전에 부피**(몸통 ~16px 맞춰 밀집 완화).
  **delta 기반**으로 부드럽게 수렴(프레임레이트 무관)하고, **접전(MELEE) 병사는 분리에서 제외**해 **제자리에서 교전**한다
  (밀림·미끄러짐 최소화). MELEE 병사는 obstacle(부피)로는 남아 CHARGE 병사가 피해 흐른다. 접전 중 지터 진폭도 축소.
- **첫 충돌 즉시 교전**(`any_engaged`) — 전원 도착을 기다리지 않는다. 사망/타격은 접전 중(MELEE) 우선.
- **자동 공방 cadence**(`_auto_strike`): 접전(MELEE) 병사는 `STRIKE_INTERVAL`(0.34s±지터)마다 자기 타겟을 향해
  주기적으로 찌른다(섬광+약한 흔들림). 병사가 **전투 내내, 그리고 복귀 대기 중에도 계속 싸우게** 해 idle 정지를 없앤다.
  타겟이 죽으면 다음 프레임 재타겟(재교전, `_retarget_all`). 살상(킬)은 Presenter가 결과대로 별도 지정.
- **펜싱 밀림**(`_auto_strike`): 공방 1회마다 **찌르는 쪽 전진 + 맞는 쪽 뒤로**를 같은 벡터(`face × FENCE_PUSH`=8px)로
  **`push_rem`에 예약**하고, `_process`가 매 프레임 이징(`PUSH_SMOOTH`)으로 소진 → **순간이동 아니라 슬라이드 애니메이션**.
  같은 벡터라 **거리 유지한 채 쌍이 좌우로 실제 이동**(`pos` 변경). 마주 선 두 병사가 번갈아 찔러 좌우로 티격태격
  밀리는 펜싱 인상. MELEE는 분리 제외라 이 밀림이 난전 내 주 위치 변화.
  (거리 유지는 근사 — 필드 가장자리 클램프 `[16,368]` 시 한쪽만 멈춰 거리가 잠깐 어긋날 수 있음. 중앙 난전이라 드묾.)
  (죽음 직전 밀림 시퀀스·공방 횟수는 **듀얼**(`kill`→`DUEL`, `_duel_count` [1,3])로 구현됨 — 아래 CLASH 참조.)
- **CLASH — 킬 스케줄**(`_build_plan`): **킬(사망) 이벤트만** 만들어(=시작−생존, 양쪽 번갈아) 재생. 총합은
  **Resolver 결과 그대로**(사망 수 불변, 테스트로 검증).
- **최소 전투시간**(`_melee_duration`): CLASH 길이를 **킬 수에서 분리**해 아래 기준으로 잡는다 — 적게 죽어도 너무
  빨리 끝나지 않게. CLASH는 이 시간 동안 지속하고(고정 간격 아님), 킬은 그 안에 고르게 분산, **남는 시간은 자동
  공방(스커미시)이 채운다**.
  ```
  deaths_a = a_start − a_final,  deaths_b = b_start − b_final
  basis    = max( min(a_start, b_start), deaths_a, deaths_b )
  melee_dur = clamp(MELEE_PER_UNIT × basis, MELEE_FLOOR, MELEE_CAP)   # 0.22 / 0.5 / 2.2(초)
  ```
  킬 재생 시각(`_schedule_times`): 등간격(`melee_dur/킬수`) 기준에서 **±`KILL_JITTER`(0.65)×간격 지터 후 정렬** —
  버스트·소강 리듬을 만들어 기계적 등간격을 깬다(킬 수·총 길이 불변, 연출 전용 RNG).
  - **min(a,b)**(동시 듀얼 수)가 바닥. **소수가 다수를 잡으면**(예: 1이 5킬) `max(deaths)`가 시간을 끌어올림.
    → 10:10 최장(2.2s cap), 1:10 즉패면 0.5s(floor), 1이 전멸시키면 2.2s.
  - **최후 전투(`_deferred_sides` + `St.FINALE`)**: `DEFER_LAST_CHANCE`(25%) 확률로 **마지막 사망 1건을 CLASH 스케줄에서 빼서 유예**.
    나머지가 다 죽고 **대형 복귀를 마친 뒤**, 필드에 남은 그 1쌍이 **개입 없는 1:1 최후 전투**로 처형된다(승패 무관).
    - **흐름**: CLASH 종료 → `stage_final_duel(loser_side)`(V=유예팀 유닛, W=적, 서로 락 + 다른 유닛의 V/W 타겟 해제 → 1:1 보장, `final` 마킹으로 복귀 제외) → `begin_retreat`(나머지만 복귀) → `St.FINALE`.
    - FINALE: `others_returned()`(최후 쌍 제외 전원 복귀)면 `fire_final_duel()` → 스테이징된 V만 **긴 밀당(3~4스텝, `DRAMATIC_STEPS`)** 듀얼로 처형. 승자 W는 상대 사망 시 `final` 해제 → 복귀. 전원 복귀하면 종료.
    - **1:1 보장**: 최후 쌍만 필드에 남고 다른 유닛의 타겟팅을 끊어 "1타겟에 다수" 개입 없음.
    - **`DEFER_DOUBLE_CHANCE`(5%, 양 팀 모두 죽음 있을 때)**: 양 팀 각 1쌍 유예 → 각자 1:1 최후 전투(레어). 동시 시작이지만 **밀당 길이를 차등**(첫 victim `DRAMATIC_STEPS` 3~4, 둘째 `DRAMATIC_STEPS2` 5~7)해 **시차 사망**(하나 먼저 쓰러지고 다른 하나가 더 버팀 → 동시 사망 어색함 제거).
    - 총합·팀별 사망 수는 불변(유예분은 FINALE에서 처리 → Resolver 결과 보존).
  - CLASH 종료: **킬 소진 && `_timer ≥ melee_dur` && 진행 중 듀얼 없음**(`duels_active`) — 마지막 듀얼이
    사망까지 끝난 뒤 복귀 시작(복귀 중 사망 방지). 남는 시간은 스커미시가 채움.
  - **kill(side) → 듀얼 밀당 → 사망**: 접전(MELEE 우선) 병사 1명을 **패자 V**로 고르고, **승자 W = V의 현재 타겟**(적).
    즉사 대신 **듀얼**을 연다 — 공방 `N = randi(1..3)`회, **N−1번 밀림 스텝**(간격 `DUEL_STEP`≈0.18s) 뒤 V 사망(`_die`:
    **넉백(Hurt 비행) → 착지 후 주저앉기(Death) → 페이드**, 섬광+흔들림 — §"병사 스프라이트" 사망 연출 참조).
    - 각 밀림 스텝 확률(`_push_kind_for`): **60% 무이동 / 30% 패자 밀림(승자 전진) / 10% 승자 밀림(패자 전진)** —
      **패자 편향이되 확정 아님**. 밀림은 (a)의 `push_rem` 재사용(거리 유지·이징 슬라이드, 밀리는 쪽이 상대 반대로 물러남).
    - **W 없으면(타겟 null/사망) 즉사 fallback**(밀림 0). `N=1`도 밀림 0 → 바로 사망.
    - V는 `DUEL` 상태: **재타겟 대상 제외·자동 공방 안 함**. HUD 병력 카운트는 **kill 발화 시 감소**(시각적 사망은 밀당 끝난 뒤).
      duel 진행 중이면 `all_returned` 미완료(복귀는 duel 끝까지 대기).
  - **재교전**: 킬로 타겟이 죽으면 다음 프레임 `_retarget_all`이 남은 적으로 자동 재교전(1:N 난전) — 전투 중 이탈은 없다.
  - **복귀는 순차(줄줄이) 이탈**: 스케줄 소진 후 `begin_retreat`(`_retreating`) → 각 병사는 **마지막 교전(유닛별 `retreat_swings`=randi[1,`RETREAT_SWINGS_MAX`=2]회 공방)**
    을 더 한 뒤 이탈한다. **단, 마지막 교전은 살아있는 적(`_foe_alive`: 타겟이 MELEE/CHARGE)이 있을 때만** — 상대가 죽는 중(DYING)이거나 이미 이탈했으면
    `retreat_swings=0`으로 즉시 복귀(**허공에 칼질=헛칼질 방지**). `_retreat_ready` 조건: CHARGE(접근만)는 즉시 / MELEE는 `retreat_swings==0` **및** `strike_t<=0` **및** `|push_rem| < RETREAT_PUSH_EPS`(1px).
    복귀 후 공방·밀림 예약은 아직 싸우는(MELEE/CHARGE) 적에게만. 마지막 교전 횟수가 유닛마다 달라
    **동시에가 아니라 순차적으로** 뒤돌아 `home`으로 복귀(`_return_step`) — "한 순간에 일괄 복귀" 어색함 제거. 완료(`all_returned`=생존자 전원 IDLE) 또는 상한에 DONE.
  - **찌르기 런지 ①은 펜싱 밀림(`_auto_strike`)으로 대체 활성**. **창 뻗기 ②·전방 슬래시 이펙트 ③은 임시 비활성**
    (코드 `[임시 비활성]` 마커, 재활성 시 주석만 해제). 공격 모션은 스프라이트 `attack` 애니, 전사는 `hurt`(넉백 비행)→`death`(착지 주저앉기) 애니가 담당. 섬광/화면 흔들림 유지.
- 패한 쪽은 **그 자리에서 전멸**(도망/복귀 없음). 생존자만 필드에 남음.
- 스킵 시 `force_result`로 즉시 최종 생존자만 남긴다.
- 큰 프레임 델타(로딩·랙)가 상태를 건너뛰지 않도록 `_process` 델타를 0.05로 상한.

### 픽셀아트 렌더링 (16비트풍)
원본은 16비트 도트 그래픽이다. 이를 위해 `Battlefield` 노드를 384×216 논리 공간에서
**안티앨리어싱 없는 `draw_rect` 픽셀로만** 그린 뒤 씬에서 `scale = ×5` 로 확대(→1920×1080)한다.
draw_rect 는 하드엣지라 확대해도 도트가 선명하다(SubViewport 불필요).
- 지형: 녹색 이끼 석조 벽돌 바닥 + 상단 성벽/구덩이 + 나무 사다리(원본 요새 맵 오마주). `_draw` 로 그린다.
- 병사: **`AnimatedSprite2D` 스프라이트**로 그린다(§아래 "병사 스프라이트"). 그림자·타격 섬광만 `_draw`.
- 하단 HUD 는 가독성을 위해 풀해상도 CanvasLayer 로 별도(3분할 금테 패널·포트레이트).

#### 병사 스프라이트 (임시 경보병 애셋)
병사는 `_units`(y-sort Node2D) 아래 병사 id별 `AnimatedSprite2D` 풀로 그린다(`_sync_sprites`).
지형(`_draw`)은 항상 뒤, 스프라이트는 y-sort 로 뒤→앞 정렬. 사망 중(DYING)도 스프라이트로 그려진다.
- **애셋**: Tiny RPG Character Pack — 팀 0=Soldier, 팀 1=Orc. `assets/units/{soldier,orc}_{walk,idle,attack,hurt,death}.png`.
  프레임 100×100(여백 포함), 실제 몸통 ≈17×22px. **개발용 플레이스홀더**(재배포 지양).
- **스케일**: 몸통 ≈17px → 바닥 타일 1칸(16px)에 맞춰 `SPRITE_SCALE = 16/17` 축소. `texture_filter=NEAREST`.
- **애니**(시트 가로 나열 → `SpriteFrames` 코드 구성):
  - `walk`(8f/10fps·루프) — 이동(`CHARGE`/`RETURN`). 생성 시 프레임 위상 랜덤(발맞춤 방지).
  - `idle`(6f/6fps·루프) — 정지(`MELEE`/`IDLE`/`DUEL`, 공격 아님).
  - `attack`(6f/18fps·1회) — `_auto_strike` 때 `atk_fire` 신호로 0프레임부터 재생. 애셋이 제자리 전방 찌르기라 펜싱과 정합.
  - `hurt`(4f/12fps·1회) — 사망 넉백 **비행 중** 자세(뒤로 젖힘).
  - `death`(4f/10fps·1회) — 착지 후 **제자리 주저앉기**(눕기).
  - `flip_h = face<0`(애셋 기본 우향).
- **정렬**: 프레임 중심(centered)에 병사 `pos` → 발끝 ≈ `pos.y+8.5`(그림자 `pos.y+8`와 일치).
- **피격 흰색 플래시**: 병사별 `ShaderMaterial`(`hit_flash.gdshader`) — `COLOR.rgb = mix(COLOR.rgb, 흰색, flash·COLOR.a)`. 진입 시 `COLOR`는 이미 (텍스처×modulate) 합성값이라 텍스처를 재샘플·재곱하지 않는다(그러면 `tex²`로 어두워짐). `flash=0`이면 기본 렌더와 픽셀 동일, 알파·사망 페이드도 그대로 보존.
  맞는 순간 `hit_t = HIT_FLASH_DUR`(0.12s) → 매 프레임 `flash = hit_t/DUR`(1→0 감쇠)로 팝. 트리거: `_auto_strike`의 타겟, 듀얼 밀당 패자(매 스텝), `_die`(치명타).
- **사망 연출(넉백→쓰러짐 하이브리드)** — 원본 롬 포물선 넉백을 유지하되 애셋과 화해:
  1. `_die` → `airborne=true`, 위로+뒤로 포물선 launch(`DEATH_VY`/`DEATH_VX`/`DEATH_GRAV`). 비행 중 `hurt` 자세. 그림자는 지면(`land_y`)에 고정 → 점프처럼.
  2. 착지(`pos.y ≥ land_y`) → `airborne=false`, `death_t=0` → `death`(주저앉기) 재생(`DEATH_COLLAPSE`≈0.4s).
  3. 이후 `DEATH_FADE`(0.35s) 동안 alpha↓ → 제거(스프라이트 풀도 정리).

## 근접 난투 교전 설정 (경보병 vs 경보병)

- 위 근접 상태머신(CHARGE→CLASH→RETREAT/FINALE)을 쓰는 교전은 **설정 화면에서 양측 경보병 + 근접**을 골라 재생한다(내부적으로 `_load_scenario(0)` 폴백과 동일 경로).
- 양측 모두 `classId 1`(경보병, at23/df21), 개활지 회피 5 — 대칭 교전이라 결과는 시드에 따라 갈린다.
- (이전 격리 더미 창기병(classId 9) vs 습격대(classId 27)는 제거됨.)

## 경궁병 사격 전투 (원거리) — 슬라이스 1

> 경궁병(원거리)/경보병(근접) 병종과 **사격 전투**를 추가한다. 위 근접 난투와 별개의 **사격 모드**다.
> 계산/연출 분리 원칙은 그대로: **Resolver가 "어느 화살이 죽이는지" 먼저 결정**하고 Presenter가 화살로 재생한다(스킵해도 결과 불변).
> 이 슬라이스는 **순수 사격 시나리오(1·3·4)**만 다룬다. 시나리오 2(사격 오프닝→근접 핸드오프)는 **슬라이스 2**.

### 전투 조합(내부 시나리오 번호)
전투 씬에는 버튼이 없다 — 아래 조합은 **설정 화면의 병종·숫자·교전 선택**으로 재생된다. 번호는 직접 진입 폴백(`_load_scenario`)과 코드 내부 참조용이다. 설정 없이 직접 씬 실행 시 **시나리오 1**을 재생. 전투 중 **키보드 아무 키 = 연출 스킵**(결과 불변), **ESC = 설정 화면**.

| # | 조합 | 방식 | 비고 |
|:--|:--|:--|:--|
| 0 | 경보병 vs 경보병 (근접) | **근접 난투**(`resolve_engagement` + CHARGE→CLASH→RETREAT) | 기존 근접 전투 재사용(양측 경보병) |
| 1 | 경궁병 → 경보병 (사격) | 사격: 공격측 **1라운드** / 방어측 0 | 경보병 **제자리 대기(passive)** — 사망만 연출 |
| 2 | 경궁병 vs 경보병 (근접) | **사격 오프닝(궁병 1발) → 근접 난투** | 보병 사상자 대부분 화살에서, 궁병은 근접 취약(거의 못 죽임) |
| 3 | 경궁병 vs 경궁병 (사격) | 사격: 양측 **1라운드** | 각 궁병 1발씩 발사 |
| 4 | 경궁병 vs 경궁병 (근거리) | = 시나리오 3 | 궁병전엔 실제 근접 없음(1발씩 사격하고 끝) |
| 5 | 영웅 vs 경보병 (근접) | **근접 난투**(`resolve_engagement` + CHARGE→CLASH→RETREAT) | 영웅 1인(HP 10)이 병사 10 몫으로 싸움 — 아래 "영웅 전투" 참조 |

- 각 궁병은 **1발**만 발사(10명이면 최대 10발). 사격은 **1회(1라운드)만 연출**한다.
- (Resolver `resolve_ranged` 자체는 다중 라운드를 지원하지만, 현재 시나리오는 모두 1라운드로 호출한다.)

### Resolver — `resolve_ranged` (순수 계산)
`resolve_ranged(rng, a, d, a_rounds, d_rounds) -> Dictionary`
- 스탯 조립·명중률은 근접과 동일(`assemble_stats`, `hit_chance_pct`) — HUD 표시용.
- 라운드 루프 `for r in max(a_rounds, d_rounds)`:
  - **슈터 수 = 라운드 시작 시점 생존자**(양측 라운드 시작값으로 잠금). `r < a_rounds`면 공격측이, `r < d_rounds`면 방어측이 각 1발.
  - 각 발사 = `_attempt_hit` 1회(근접과 같은 회피→특수 게이트→데미지). troops factor = 슈터 병사수(라운드 시작).
  - **명중(HIT) → 상대 1명 감소**. 상대 생존자 초과 킬 불가(**cap**, 초과 화살은 `kill=false`).
  - 볼리 순서: 같은 라운드 안에서 공격측 볼리 → 방어측 볼리(슈터 수는 라운드 시작값 고정 = "동시" 근사).
- 반환:
  ```
  {
    shots: [ {side, round, kill}, ... ],   # 발사 순서대로
    final_a_soldiers, final_d_soldiers,
    stats_a, stats_d,                      # 근접(resolve_engagement)과 동일 구조(HUD)
  }
  ```
- side별 `kill==true` 개수 = 상대 사망 수 = 시작−최종(불변식).

### Presenter — 사격 상태머신
- **setup**: 양측을 **진형(`home`)에 바로 배치**하고 중앙(적)을 바라본 채 대기. 돌격·분리·복귀 없음(사격은 진형 유지).
- **샷 스케줄**: `shots`를 라운드 순서로 시간에 깔고(라운드 내 스태거 + 라운드 간 간격) 재생. 각 shot마다:
  1. 슈터(해당 side 생존자 중 하나)가 **활 당기기 애니**.
  2. 릴리스 프레임에 **화살 발사체** 생성 → **예측 지점**(리드)으로 **고정 포물선** 비행.
  3. **착탄** 시 임팩트 섬광 + `kill`이면 **그 화살이 꽂히는 순간** 타겟 사망(`_die` 넉백 연출 재사용).
  - **킬 샷은 서로 다른 생존 적**에 배정(분산), 논킬은 임의 생존 적(빗맞음/스침).
  - **사망 시점 = 착탄 순간**(발사 순서·비행 시간에 따라 개별 시차). 단, **이번 라운드에 아직 발사할 슈터**가 맞으면
    발사를 마친 뒤 쓰러지도록 유예(`death_queued`) — 발사하지 않는 side(예: 시나리오1 경보병)는 유예 없이 **즉시 사망**.
    유예 판정은 `begin_shot_round(사격하는 side들)`이 만든 **슈터 풀 소속 여부**로 한다(사격 안 하는 side는 풀에서 제외).
- **HUD 병력**: 매 프레임 `alive_count(side)` 폴링으로 갱신 → 화살 착탄 순간 자연 감소(발사≠사망 시점 분리).
- **완료**: 모든 화살 발사·착탄·사망 연출 종료 → 결과(POST) → DONE. **복귀 단계 없음**.
- **스킵**: 키보드 아무 키 = `force_result`로 즉시 최종 생존자. ESC = 설정 화면.

### 전장 렌더 — 활·화살 (연출 전용)
- **활 당기기 애니**: `assets/units/archer_bow.png`(Soldier_Attack03, 900×100 = **9프레임**, 1회). 릴리스 프레임(≈6/9)에 화살 loose.
- **화살 발사체**: `assets/units/arrow.png`(Arrow01 32×32, NEAREST). **예측 요격 + 고정 포물선**(유도 아님): 발사 시 타겟의 돌격 속도(`_charge_velocity`, X우선 근사)와 화살 속도 `ARROW_VX`로 **정확한 요격 시각 T를 2차 방정식으로 풀어**(`_predict_intercept`, `|R+V·T| = ARROW_VX·T` → 최소 양수근) 만나는 지점을 겨냥 → **화살과 적이 그 지점에서 딱 만난다**(서로 접근해도 정확). 수평 등속 + 수직 중력 g(정점 `_arrow_peak` = `ARROW_ARC_PEAK`, **단 궁병 vs 근접(스커미시)은 절반으로 완만**), 접선 회전. `t≥flight` 착탄 시 임팩트 섬광(살상이면 그 순간 사망). **명중(`is_kill`) 화살은 임팩트 후 소멸**, **빗나간 화살은 땅에 박혀 잔류**(아래). 정적 타겟/요격 불가 시 현재 위치.
- **빗나간 화살 잔류(박힘)**: 착탄한 화살 중 **`is_kill=false`(빗나감·비살상)** 는 소멸하지 않고 **땅에 박힌 채 남는다**. 연출은 착탄 순간 고정:
  - **산포**: 빗나간 화살은 조준점 그대로가 아니라 **목표 주변에 흩뿌려** 박힌다. `_spawn_arrow`에서 `is_kill=false` 화살의 착탄 목표(`to`)에 `_miss_scatter()` 를 더한다 — **x는 좌우(±`ARROW_MISS_SCATTER_X`)**, **y는 아래(+`ARROW_MISS_SCATTER_Y`)로만**.
  - **각도**: 화살촉(진행 방향 끝)이 **아래를 향하도록** 수직(PI/2)에서 **진행 방향으로 소폭 틀어**(`STUCK_TILT` 범위, `_rng`로 발마다 약간 변동) 세운다 → 오늬(꽁지)는 쏜 쪽으로 살짝 기욺.
  - **크롭**: 땅에 묻힌 **화살촉 쪽(진행 방향 끝)** 을 `region_rect`로 잘라내(`STUCK_KEEP` 비율만 노출) 오늬~샤프트만 지면 위로 보이게 한다.
  - **정렬(z)**: 박힌 화살은 `_arrows_node`(병사 위)에서 **`_units`(y-sort)로 옮겨** **병사 발끝과 화살 착탄 지점을 같은 기준으로 앞뒤 정렬**한다. 병사 스프라이트는 원점이 발밑(그림자)보다 `GROUND_SORT_DY`(8px) 위이므로, 화살도 **origin(정렬 기준 = `position.y`)을 착탄 지면보다 `GROUND_SORT_DY` 위**에 두고, 묻힌 화살촉은 `offset`(회전·스케일 역변환)으로 **실제 지면에 닿게** 보정한다. → 앞(아래)에 선 병사는 화살을 가리고, 뒤 병사는 화살에 가려진다.
  - **잔류**: 임팩트 섬광은 그대로 내되 `queue_free()` 대신 `_stuck_arrows`에 보관 → **전투 끝까지 누적**.
  - **정리**: 전투 초기화(`setup*` → `_clear_arrows`) 시 비행 화살과 함께 박힌 화살도 모두 제거.
- **경궁병 스프라이트**: 병사(Soldier) walk/idle/hurt/death + **bow(당기기)**. 팀1 궁병은 `modulate` 색조로 구분(임시 플레이스홀더). 경보병(시나리오1 방어측)은 기존 Orc idle(제자리 대기).

### 시나리오 2 — 경궁병 vs 경보병 (사격 + 돌격, 실시간 동시)
**계산은 두 순수 함수를 연쇄**(사격 오프닝 → 근접)하되, **연출은 사격과 돌격을 동시에** 재생한다.

- **경보병(돌격)**: 근접 vs 근접처럼 **화면 밖에서 즉시 돌격**(off-screen 스폰 → CHARGE). 처음부터 궁병 진영으로 달려온다.
- **경궁병(사격→제자리 대기→마중→가담)**: 자기 **진형에 서서(aiming) 먼저 사격**하고, 쏜 뒤엔 **전진하지 않고 제자리에서 대기**(`hold_ground`). 적이 `MEET_RANGE`(≈40px) 안으로 다가오면 **한두걸음만 마중 나가**(`hold_ground` 해제 → 짧은 접근) 근접으로 **반격**한다. **교전이 시작되면(어느 한쪽이라도 접전=`clash_on`) 대기 중이던 뒷열도 즉시 대기 해제 → 다 같이 근접 가담**(소수 적을 다수 궁병이 에워쌈, 뒷열이 놀지 않음). 사격·돌격이 **한 필드에서 실시간 병렬**.
- 사격은 움직이는 적을 노리므로 **예측 요격**: 화살·타겟 속도로 만나는 시각/지점을 2차 방정식으로 풀어 **그 지점으로 고정 포물선**을 쏜다(유도 아님, 화살과 적이 딱 만남).

계산:
1. **사격 오프닝**: `resolve_ranged(rng, 궁병, 보병, 1, 0)` — 궁병 각 1발, 보병 **반격(사격) 없음**. 단 **돌격 중 보병은 화살 회피 ↑**(`SCENARIO2_CHARGE_EVASION`=25, 볼리 명중을 낮춤) → 절반 이하만 전사하고 나머지는 근접까지 도달(착탄마다 즉시 사망). 근접 회피는 기본(5).
2. **근접 난투**: 사격 생존자로 `resolve_engagement`. **궁병 근접 취약 = 병종 상성**(보병>궁병): 보병은 궁병 상대로 **공격 +4 / 방어 +2**, 궁병은 보정 없음. 별도 at/df 페널티는 없다. 이 상성은 **사격에도 적용**(보병 방어 +2 → 볼리도 덜 치명적).
   → **5:5 순수 근접이면 보병 우세**. 궁병의 이점은 무피해 볼리뿐.
- **결과 경향**(다수 시드): 볼리로 보병 ~2명 사망 → 근접에서 보병이 상성 반격 → 최종 궁/보 ≈3.4/3.9, **보병 우세**(보병승 ~50% / 궁병승 ~37% / 무 ~11%). 볼리로 많이 줄인 판은 궁병이 수적 승, 다수가 근접 도달한 판은 보병 압도(전술 분기).
   - 근접 시작 인원 = **궁병 10 vs 보병(사격 생존자)**. 킬 스케줄(`_build_plan`)·최소 전투시간(`_melee_duration`)은 실제 시작 인원(`_melee_start_a`/`_melee_start_b`)으로 계산.
   - **빠른 근접**(`_fast_melee` / `set_fast_melee`): 궁병 근접전은 근접 vs 근접보다 **짧게 끝난다** — CLASH 길이 축소(`FAST_MELEE_*` = 0.26/0.6/1.5, 일반 0.22/0.5/2.2), 최후 1:1 유예(FINALE) 생략, **킬 시 듀얼 밀당 0~1스텝(짧게)**, 복귀 전 마지막 교전(retreat swings) 0(뒤 헛칼질 제거). 복귀 이동 속도는 일반과 동일.
3. **상태 전이**: 사격 볼리(St.RANGED, 배경에선 이미 돌격 중)는 **화살이 다 착탄하면 즉시**(죽음 애니 대기 없이, `_then_melee`→`arrows_in_flight` 기준) **재스폰 없이** 근접(St.CHARGE→CLASH→RETREAT)으로 이어진다(초반 헛칼질 겹침 최소화, 병사는 계속 필드에서 이동·교전). 궁병 근접 시 활 대신 **검 공격 모션**. 보병이 화살만으로 전멸(생존 0)하면 근접 없이 결과.
   - **AT/DF 표시 연속성**: 순수 근접(시나리오 0) CHARGE는 지휘보정을 base→조립값으로 **틱업**(`_atdf_tick_t` = `_timer/STEP`)하지만, **시나리오 2 근접은 사격 단계에서 이미 조립값을 보였으므로 즉시 조립값(t=1)** 을 유지한다(`_fast_melee` 게이트). base로 떨어졌다 되오르는 어색함 방지 — 같은 부대·같은 버프의 연속 전투이므로.
- **순 결과**: 보병 전사자는 **대부분 화살**(돌격 중), 궁병 전사자는 **근접**에서.

## 영웅 전투 (시나리오 5 — 영웅 vs 경보병 근접)

> **영웅(지휘관) 1인이 병사 여럿을 상대하는 근접전.** 계산은 기존 근접(`resolve_engagement`)을 **그대로** 쓰고,
> 연출만 "1스프라이트 + HP"로 바꾼다. 별도 전투 공식·상태머신을 새로 만들지 않는다.

- **매치업**: 영웅(`classId 4`, base **at 27 / df 24**) 1인 vs 경보병(`classId 1`) 10인.
- **"10명 몫"**: 영웅은 화면엔 **1스프라이트**지만 전투 계산상 **병사 10명 유닛**으로 취급한다 —
  `make_unit(4, side, 10, …)` 이라 **공격 13회**(`attack_count=병사수+3`)·데미지 factor도 10 병력 기준.
  즉 근접 결과(`final_a_soldiers`)가 곧 **영웅의 잔여 HP(0~10)**다.
- **자기 지휘보정 없음(`self_cmd=false`)**: 단독 영웅은 지휘할 부대가 없으므로 자기 지휘보정을 받지 않는다 →
  표시·계산 스탯이 **27/24로 유지**(class 4의 cmd +2/+4를 받으면 29/28이 되어 경보병 at 25가 df를 못 뚫어 **영웅이 무피해**가 됨).
  27/24면 경보병이 영웅에 피해를 줄 수 있어(무피해 아님) HP 연출이 성립하고 1:다 스케일 균형이 맞는다.
  - **결과 경향**(다수 시드): 영웅 평균 잔여 HP ≈ **3.9/10**, 영웅이 피해 입는 판 ≈ **99.7%**, 영웅 사망 ≈ **7.5%**,
    보병 전멸 ≈ **23%**. 영웅이 대개 이기되 피 흘리고, 물량에 쓰러지기도 한다.
- **병종(`kind`)**: 영웅은 `""`(상성 중립) — 경보병에 우열 없음. 경보병은 `"infantry"`.
- **연출**: 기존 근접 상태머신(CHARGE→CLASH→RETREAT)을 재사용. 영웅은 `setup_hero(hp, infantry_n)`로 진영 세로 중앙에
  1인 스폰(3행 진형 대신 단독 배치), 나머지는 경보병 10인 근접과 동일.
  - **영웅 HP 피격**(`Battlefield.kill(0)`): 영웅이 접전 병사로 뽑히면 **죽지 않고** `hp -= 1` + **비살상 피격 리액션**
    (`_hero_recoil`: 흰색 플래시 + 흔들림) 후 계속 교전. **`hp`가 1→0이 되는 마지막 1발에서만** 실제
    사망(듀얼→넉백→쓰러짐, 일반 병사와 동일). HUD 병력 숫자(좌측)가 영웅 HP(10→0) 역할을 한다.
  - **영웅 제자리 사수**: 영웅은 보병에게 **뒤로 밀리지 않는다** — 접전 펜싱 밀림(`_auto_strike`)·듀얼 밀당(`_apply_duel_push`)·
    피격 리액션 모두 영웅에는 `push_rem`을 누적하지 않는다(보병만 밀림). 영웅은 진영에 버티고 보병이 에워싸 부딪는 그림.
  - **`force_result` 트림 제외**(`_reduce_to`): 영웅 side는 HP를 kill 이벤트로 처리하므로 스프라이트 수를 `final_n`으로
    트림하지 않는다(1스프라이트 유지, 사망은 kill 이벤트가 담당).
  - **최후 1:1 유예(FINALE) 생략**(`_hero_battle`): 영웅은 1인이라 "나머지 다 죽고 최후 1쌍" 그림이 부적합 → `_build_plan`에서 유예 스킵.
- **HUD**: 좌측 "영웅"(HP 숫자·AT/DF 27/24), 우측 "경보병". 영웅 AT/DF는 base==조립(27/24)이라 CHARGE 틱업해도 그대로.

### 병종 상성 (기·보·창·궁 가위바위보) — `LangResolver._lang_type_bonus`
유닛의 **병종(`kind`)** 으로 상성을 판정한다. **상성 우위면 공격 +4 / 방어 +2**(`TYPE_ADV`, 원작 스타일), 우위가 아니면 보정 없음(지는 쪽에 별도 보정은 주지 않는다). `assemble_stats`가 조립 at/df에 가산하며 **사격·근접 모두** 적용된다.

상성표(A가 B를 이기면 A에 +4/+2):

| 이기는 쪽 | 지는 쪽 |
|:--|:--|
| 기병(cavalry) | 보병, 궁병 |
| 보병(infantry) | 창병, 궁병 |
| 창병(spear) | 기병, 궁병 |
| 궁병(archer) | *(없음)* |

- **사이클**: 기병 > 보병 > 창병 > 기병.
- **궁병은 원거리 이점의 대가로 근접에서 모든 병종에 약함**(누구에게도 상성 우위 없음 → 기·보·창 모두 궁병에 +4/+2).
- 현재 구현된 병종은 **경보병(infantry)·경궁병(archer)** 뿐. 기병·창병은 데이터·연출 준비되면 같은 표로 확장.
- ROM `matchup.csv`(magicTier 기반, §2.3)와 별개의 lang_battle 자체 표다(더미가 같은 classId라 ROM 상성은 0).

### 미구현 / 유예 (사실 표기)
- 화살은 **예측 요격 + 고정 포물선**(2차 방정식, 유도 아님). 타겟 속도는 X우선 등속 **근사**(정밀한 대각/감속 예측·바람·풍향 미구현).
- 궁병 근접 전환 시 사거리·재사격 판단(다가오는 적에게 추가 사격) 없음 — 오프닝 1발 후 바로 근접.
- 경궁병/경보병 스탯은 ROM 클래스 **근사 매핑** — 정확한 원작 궁병 수치는 미반영.

### 테스트 시나리오 — 최소 전투시간 (`_melee_duration`)

- [경계] 1:10, 1 즉패(생존 0/10, deaths 1/0) → basis=max(1,1,0)=1 → **0.5s**(floor). 대부분 케이스.
