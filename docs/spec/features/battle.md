# Feature: Battle (전투씬 · 개시 · 복귀)

> 스크립트: `scenes/combat/battle.gd` (`extends CanvasLayer`) · `scenes/combat/battle_field.gd` (`class_name BattleField extends RefCounted`) · `scenes/game/game.gd`

플레이어 부대가 인접한 적을 공격하면 **게임 내 오버레이 전투씬**이 열려, 양 부대원이 실시간으로 교전하는 모습을 관전한다. 판정은 [CombatResolver](combat.md)를 재사용한다. 전투가 끝나면 사상자를 부대에 반영하고 오버레이를 닫는다.

이 문서는 **슬라이스 2**(전투씬·개시·복귀)다. 판정 수학은 [combat.md](combat.md).

## 개시 (`game.gd` + 행동 메뉴)

공격은 [행동 메뉴](party-action-menu.md)로 개시한다(근접 무기 월드맵 사거리는 0이라 붙어야 침).

- **근접 `[공격]`**(공격 가능 적 클릭 팝업): 그 적에 **인접한 도달 칸으로 자동 이동**(이동 애니메이션)한 뒤 근접 전투. **승리 시(수비 전멸·공격 생존) 공격 부대가 수비 타일로 이동**한다(점령).
- **사격 `[사격]`**(중앙 SHOOT 모드 또는 적 팝업): **현재 위치**에서 원거리 전투(이동·점령 없음). 사거리(활 3·완드 2) 내 적만.
- 이동을 했으면 `mark_moved`, 개시 시 `mark_attacked`로 행동을 끝낸다. `ClickRouter`는 `MOVE` 모드의 빈칸/건물/부대 클릭만 처리(`ATTACK` 제거).
- **공격측 = 플레이어 부대, 방어측 = 클릭한 NPC 부대.** (NPC가 먼저 거는 전투는 미구현 — NPC AI는 이동만.)
- 전투 중에는 월드맵 좌클릭·턴 종료를 잠근다(`_in_battle`).

## NPC가 거는 전투 (공격 페이즈)

NPC [이동 페이즈](npc-movement.md)가 끝난 뒤 **공격 페이즈**가 이어진다.

- 각 NPC(살아있고 이번 턴 아직 공격 안 함)가 **공격 가능 범위 안의 적 부대**(플레이어 또는 다른 NPC)가 있으면 전투를 건다. 범위 = `max(attack_range(), 1)` — **근접(사거리 0)은 인접(1)까지, 원거리는 사거리까지**(`_adjacent_enemy`). NPC 순서대로 하나씩 처리한다.
- 공격을 건 NPC는 `mark_attacked`로 이번 턴 행동을 끝낸다. 범위 내 적이 없으면 넘어간다.
- 전투 중 사망으로 목록이 바뀌므로, 처리 중 제거된(전멸) 부대는 건너뛴다.
- 이동 애니메이션과 겹치지 않게 **이동 페이즈 완료 후** 공격 페이즈를 돈다. 도중 새 턴이 시작되면(세대 가드) 중단한다.

## 전투 모드: 교전 거리(`distance`)

전투는 개시 시 **교전 거리**(`distance`, 두 부대 사이 헥스 거리)를 갖는다. 이 거리로 어떤 유닛이 싸우는지 정해진다(기획 원본 "공격 유형"). 오버레이(`battle.gd`)·헤드리스(`BattleSim`)가 같은 규칙을 쓴다.

- **근접(`distance ≤ 1`, 인접)**: 전원 행동 — 근접 유닛은 돌격, 원거리 유닛은 사격.
- **원거리(`distance ≥ 2`)**: **자기 무기 사거리 ≥ `distance`인 유닛만 사격**하고, 그렇지 못한 유닛(근접 무기 포함)은 정지·공격 불가(너무 멀어 닿지 않음). 양측 동일 — 사거리가 닿는 유닛만 상호 사격, 나머지는 대상은 되나 반격 못 함.
  - 예: 거리 3 교전 → 활(사거리 3)은 사격, 완드(2)·근접(0)은 대기. 거리 2 → 활·완드 사격, 근접 대기.
- 내부적으로 `ranged := distance >= 2`(전투 시간·2배속·무기 선택은 이 파생값이 그대로 좌우). 기존 이진 `ranged_mode`(사거리 `< 2` 게이트)를 **거리 게이트(`< distance`)로 일반화**한 것 — 거리 2 교전은 종전과 동일하고, 거리 3+에서만 짧은 사거리 무기가 걸러진다.
- `game.gd`가 개시 시 공격자·대상의 **헥스 거리**를 구해 전달한다(근접 [공격] = 1, [사격] = 부대↔표적 헥스 거리).
- **[투석] 전투**([공성병기](siege-engines.md#battlegd-통합-전투--투석기구조물-전투원)): `start(attacker, defender, distance, include_siege, wall)` — `include_siege`면 양 부대 투석기를 전투원으로, `wall`(성벽 거점)이 주어지면 **성벽을 구조물 전투원(`structure`)으로 방어 팀에 스폰**한다(이때 `defender`는 `null` 허용 — 성벽만 방어). 구조물 전투원은 이동·공격·상태이상이 없고 투석 flat 피해만 받으며, 전투 종료 시 남은 hp가 `building.wall_hp`에 반영된다.
- **한계(미구현)**: 원거리 유닛은 `ItemTypes.active_weapon`이 고른 무기 하나로 싸운다 — `ranged_weapon`이 **사거리 ≥ 2인 *첫* 무기**를 고르므로(최대 사거리가 아님), 한 유닛이 원거리 무기를 **둘 이상**(예: 완드 2 + 활 3) 들면 거리 3에서 짧은 쪽(완드)을 골라 대기할 수 있다. 현재 카탈로그엔 그런 유닛이 없어 미발현. 최대 사거리 무기 선택은 후속.

## 전투 재생 분기: 오버레이 vs 헤드리스

- **플레이어가 참여하는 전투**(플레이어가 개시했거나 NPC가 플레이어를 공격) → **오버레이로 관전**한다(`_run_battle`, 아래). 공격 페이즈에서는 순차로 `await`한다.
- **NPC ↔ NPC 전투** → **화면 없이 즉시 결산**한다(`BattleSim.resolve_battle`, 아래). 플레이어가 보지 않는 전투라 위치·이동을 생략한 근사 결산이다. **투석기를 실은 부대끼리면 투석 볼리도 함께 결산**한다(NPC↔NPC 투석기 결투 — 5g-B, `_resolve_battle_headless`가 양측 `siege_units` 전달·전투 후 `prune_destroyed_siege`).
- 두 경로 모두 결과(생존자)를 `_apply_survivors`로 반영한다.

## 헤드리스 전투 결산 (`battle_sim.gd`, 순수)

> `class_name BattleSim extends RefCounted`

오버레이 없이 두 부대의 교전 결과만 계산하는 순수 함수(테스트 용이). 위치·이동·리치·투척을 무시하는 **시간 기반 추상 결산**이라 오버레이의 공간 전투와 결과가 다를 수 있다(플레이어가 안 보는 NPC끼리 전투에만 쓴다).

- `BATTLE_TIME = 10.0` — 한 전투 지속 시간(초).
- `resolve_battle(a_members, b_members, rng, distance := 1, a_siege := [], b_siege := []) -> Dictionary` — `{a: 생존 human 목록, b: 생존 human 목록}`. `distance` = 교전 헥스 거리(기본 1 = 근접). 내부 `ranged := distance >= 2`. `a_siege`·`b_siege`([SiegeUnit](siege-engines.md) 목록, 기본 빈 배열)이 있으면 **멤버 전투 전에 투석 볼리**를 먼저 정산(아래).
  - 각 멤버를 유닛(`{human, team, hp=hit_points, alive, weapon, can_attack, interval, next_t, effects}`)으로 만든다. `weapon` = `ItemTypes.active_weapon(멤버.weapons, ranged)`(근접=주무기, 원거리=활), `interval` = `CombatResolver.attack_interval(멤버, weapon)`(공격 간격 초). `effects` = `{}`([상태이상](status-effects.md)).
  - **시간 기반 이산 시뮬**: 다음 공격 시점(`next_t`)이 가장 이른 유닛부터 처리한다. 그 유닛이 상대 팀의 살아있는 유닛 하나를 [1회 공격](combat.md)(`resolve_hit`)하고, `next_t += interval`. `next_t`가 `BATTLE_TIME`을 넘으면 종료. 즉 각 유닛은 10초 동안 `10 ÷ interval`회쯤 공격한다.
  - **[상태이상](status-effects.md)**: 이벤트 사이 경과 시간만큼 모든 유닛에 `StatusEffects.advance`로 **출혈 도트**를 hp에서 차감한다(0 이하면 사망). 공격 차례가 온 유닛이 **기절**(`is_stunned`) 상태면 공격을 건너뛴다. `resolve_hit`의 `inflict`가 있으면 피해 적용 후 대상에 `StatusEffects.apply`. 시간 만료로 끝나면 마지막 이벤트~종료 사이 잔여 도트를 적용하지만, **한 팀이 전멸해 끝나면 그 시점이 종료라 이후 도트는 없다**(오버레이가 전멸 즉시 `_finish`하는 것과 동일).
  - **원거리 교전**(`distance ≥ 2`): 활성 무기 **사거리 ≥ `distance`인 유닛만 `can_attack=true`**. 사거리가 못 미치는 유닛(원거리 무기 없음 포함)은 `can_attack=false`(대상은 되나 반격 못 함). 근접 교전(`distance ≤ 1`)은 전원 `can_attack=true`.
  - **위치·리치·투척은 무시**한다(근사) — 대상은 목록 앞쪽의 살아있는 적, 투척 무기 유닛도 활성(주무기/활)으로만 공격. 공간 재현은 오버레이 몫. NPC끼리 전투에만 쓰므로 허용.
  - `RandomNumberGenerator`로 결정적. **단, 생존자 `Human.hit_points`를 최종 hp로 덮어써 반영한다(부작용)** — 같은 Human 배열을 재사용하면 두 번째 호출은 이미 깎인 hp에서 시작하므로, 결정성 확인 시 입력을 매번 새로 만든다.
  - **투석 볼리 프리페이즈**(공성 유닛 있을 때, NPC↔NPC 투석기 결투 — 5g-B): 멤버 이산 시뮬에 앞서 양측 투석기가 **전투당 1발**씩 쏜다(오버레이 전투당 1발 규칙과 동일). 각 투석기는 `Siege.in_fire_band(distance, min_range, fire_range)`(밴드 4~5)일 때만 발사 — 근접(거리 ≤3)이면 아예 안 쏜다. **동시 볼리**: 양측이 **볼리 전 상태를 스냅샷**해 표적을 고르므로 한쪽이 먼저 파괴돼도 반격 1발은 보장된다(상호 반격). 표적 = `BattleSim.bombard_pick(적 투석기, 적 멤버, MAX_BOMBARD_TARGETS)` — **적 투석기 우선 → 적 멤버, 최대 5**(비공간이라 목록 앞쪽부터, 거리 정렬 없음). 표적별 `Siege.hit_succeeds(rng.randf(), CATAPULT_HIT_CHANCE=0.1)` 명중 판정 → 명중 시 `Siege.rolled_damage(attack, rng.randf())` flat 피해(방어구·회피·상성 무시)를 표적 `hit_points`에서 차감. 볼리 뒤 죽은 멤버·파괴 투석기(`hit_points ≤ 0`)는 이후 멤버 시뮬에서 빠진다.
  - **공성 유닛 지속**: 투석기 `SiegeUnit.hit_points`를 in-place로 깎아 반영(멤버 hp 지속과 동일). 파괴 투석기 제거(`prune_destroyed_siege`)는 호출부(`game.gd._resolve_battle_headless`) 몫. 승패는 **멤버 기준** — 투석기는 승패에 안 셈(오버레이 `team_wiped`와 동일). 거리 4~5 교전은 근접 무기 사거리 부족으로 멤버가 대개 대기라 **사실상 투석 결투**다.

## 전투 오버레이 (`battle.gd`, 관전 전용)

- 월드맵 위를 어둡게 덮는 `CanvasLayer`(높은 layer). 플레이어 입력을 받지 않는다(관전).
- **화면 밖 돌격 스폰**(근접 공격자): **근접 교전(`distance ≤ 1`)의 근접 유닛**(활성 무기 사거리 < 2)은 **화면 밖**(공격측 왼쪽·방어측 오른쪽, 전투 라인에서 `CHARGE_OFFSET`(80px) 밖)에서 스폰해, **라인에 멈추지 않고 곧장 최근접 적을 향해 연속 돌격**한다(별도 입장 페이즈 없이 실시간 시뮬의 접근 로직이 그대로 처리). **궁수·완드 등 원거리 유닛(사거리 ≥ 2)** 도 **화면 밖에서 진입**해 **대열 슬롯으로 이동한 뒤 자리잡고(정지) 사격**한다(아래 [원거리 대열 진입](#랑그릿사1md식-연출--액자--hud--템포)). 원거리 교전(`distance ≥ 2`)에선 **근접 유닛도 같은 계단식 대열로 정렬**해(사거리 미달로 공격은 못 하지만 대형으로 서서 사격당함) — 즉 원거리 교전은 전원 대열, 근접 교전은 궁수만 대열·근접은 돌격. 유닛별 이동속도가 제각각(±20%)이라 부대가 우르르 시차를 두고 몰려온다. ([투석] 전투는 [순차 연출](#투석-오버레이-순차-연출).)
- **이동속도 랜덤 보정**: 각 멤버의 이동속도는 `UNIT_SPEED`(520px/s)에 **전투 시작 시 1회 굴린 ±`SPEED_JITTER`(20%)** 를 곱한 값(`speed`, 유닛별 고정)으로 접근·돌격한다. **공격속도(공격 간격)에는 적용하지 않는다** — 전투 결과 변동을 최소화하기 위해 이동에만 보정을 둔다.
- **개별 병사 렌더(랑그릿사식 10:10)** — 팀 멤버를 **각자 토큰 1개씩** 그린다. 공격측은 화면 좌측 세로 열, 방어측은 우측 세로 열. 판정·시뮬(멤버별 hp·`resolve_hit`·크리티컬·상태이상·타겟팅·이동·생존/사망)이 멤버 단위이고, 렌더도 멤버 단위로 맞춘다(팀 묶음 토큰 없음).
  - **병사 토큰**(`_make_token`) = 팀색 몸통 + **hp 숫자 라벨** + **작은 HP 바**(현재 hp ÷ `max_hp`, 초록 채움). 상태이상 tint: **기절=회색·출혈=붉은색**.
  - 토큰 위치 = 그 멤버의 시뮬 위치(`pos`) — 세로 열로 스폰돼 접근·교전 시 각자 움직인다. 죽으면 [사망 넉백](combat-feedback.md) 후 페이드.
  - **투석기·성벽/성문 토큰**도 각자 자기 토큰(멤버와 같은 개별 렌더).
- 전투는 **근접 모드 `BATTLE_TIME = 10초` / 원거리 모드 `RANGED_BATTLE_TIME = 5초`** 동안 진행된다. 유닛마다 **쿨다운**이 있어, 0이 되면 대상에 1회 공격하고 쿨다운을 그 무기의 **공격 간격**(`CombatResolver.attack_interval`)으로 리셋한다. 민첩이 높을수록 공격이 잦다.
- **원거리 모드 2배속 재생**: 원거리 전투는 **공격 간격 × 0.5(2배 빠름) + 전투시간 5초**로, 근접 10초 전투와 **같은 교전 내용을 절반 시간에** 보여준다(발사 횟수는 동일). 헤드리스 [BattleSim](#헤드리스-전투-결산-battle_simgd-순수)은 결과가 같아 변경하지 않는다(10초·1배 = 5초·2배 발사 수 동일).
- 멤버 유닛 상태: `{human, team, hp, max_hp, pos, speed, alive, cooldown, weapon, range, melee_reach, throw, throws, throw_reach, node, body, hp_label, hp_fill, color, effects}`. `node`=그 멤버 토큰, `body`=몸통, `hp_label`=hp 숫자, `hp_fill`=HP 바 채움 사각, `max_hp`=HP 바 기준(`human.max_hp()`), `color`=팀색. `pos`=현재 위치(근접 교전의 근접 유닛은 화면 밖에서 시작해 적을 향해 돌격, 그 외는 사격 라인에서 시작). `speed`=이 유닛 이동속도 = `UNIT_SPEED × (1 ± SPEED_JITTER 랜덤)`. `effects` = 상태이상 상태(`{}`). `weapon` = `active_weapon(멤버.weapons, ranged)`(`ranged = distance >= 2`), `range` = 그 무기 공격거리. `melee_reach` = `weapon_reach(weapon) × MELEE_REACH_PX`(근접 공격 개시 거리 — **리치 긴 무기가 더 멀리서 = 선제**). `throw` = `throwing_weapon(멤버.weapons)`, `throws` = 투척 횟수, `throw_reach` = `melee_reach + throw_range × THROW_PX`.
- **실시간 진행**(`_process`, delta) — 살아있는 각 유닛(원거리 교전에서 **사거리 < `distance`인 유닛은 정지** — 근접 무기 포함):
  1. 쿨다운 감소. 대상 = **가장 가까운 살아있는 적**(`BattleField.nearest_enemy`, 매 프레임 재탐색 → 상대가 죽으면 새 상대).
  2. **원거리**(range ≥ 2): **이동 없이 제자리**에서 쿨다운마다 사격(`resolve_hit`, 일방, 투사체 연출).
     - **근접 전환(근접 모드 한정)**: 최근접 적이 `CHARGE_RANGE_PX`(초안 120px) 안으로 들어오면 그 유닛을 **근접 교전으로 전환**한다 — `weapon`을 `melee_weapon`(없으면 현재 무기=활)으로, `range`를 1로 바꿔 이후 프레임부터 접근·근접 공격(3번 분기)한다. 한 번 전환되면 되돌리지 않는다. (원거리 모드에서는 근접 유닛이 얼어 있어 접근하는 적이 없으므로 전환도 없다.)
  3. **근접/투척**(range 1): 대상 쪽으로 그 유닛 `speed`(=`UNIT_SPEED × ±20%`)만큼 접근하며 —
     - **투척**: `throw`가 있고 `throws < MAX_THROWS`(3)이며 거리가 `melee_reach`~`throw_reach` 사이면 쿨다운마다 투척(`resolve_hit`, 투척 무기, 일방, 투사체). `throws++`.
     - **근접**: 거리가 `melee_reach` 이내면 쿨다운마다 근접 공격(`resolve_hit`, 주무기). 아니면 계속 접근.
     - 즉 투척 유닛은 **접근 중 최대 3회 투척 → 근접 리치 도달 시 주무기 근접**으로 전환한다.
  - 멤버 공방 연출은 **멤버 단위**다 — `_attack`은 **공격 멤버가 표적 멤버로 돌진(lunge)**, 대미지 숫자·피격 반짝임은 **표적 멤버 토큰**에, 원거리/투척은 **멤버→멤버 투사체**로 낸다. 매 프레임 각 멤버 `_sync_node`가 위치·hp 숫자·HP 바·상태 tint를 갱신한다.
  - 0 이하가 된 **멤버**는 [사망 넉백](combat-feedback.md) 후 페이드한다(개별 토큰). 투석기·성벽 등도 동일.
  - **[상태이상](status-effects.md)**: 매 프레임 `StatusEffects.advance(effects, delta)`로 출혈 피해를 hp에서 차감(0 이하면 사망). **기절**(`is_stunned`) 유닛은 그 프레임 공격을 건너뛴다. `_attack`에서 `resolve_hit`의 `inflict`가 있으면 대상에 `StatusEffects.apply`. 출혈 도트 피해는 **붉은 숫자**로 그 멤버 위치에 뜬다. 기절·출혈은 멤버 토큰 **tint**(회색·붉은색)로도 표시(`_sync_node`).
- **투사체 속도·궤적**: 화살·투창은 `PROJECTILE_TIME`(0.24초, 기존의 1/2 속도) 동안 대상까지 **포물선**(아치 높이 `PROJECTILE_ARC` 40px, 중간 지점 최고)으로 난다(피해는 발사 즉시 적용, 연출만). **[투석] 볼리 투사체는 `SIEGE_PROJECTILE_TIME`(0.72초, 화살의 1/3 속도)** 으로 더 느리게 날고, 피해도 **착탄 시점**에 적용한다(아래 [투석 순차 연출](#투석-오버레이-순차-연출)).
- **종료**: 전투시간(근접 10초/원거리 5초)이 지나거나 한 팀이 전멸하면 종료한다. **종료 시점에 아직 날아가는 투사체가 남아 있으면, 마지막 투사체가 착탄할 때까지 기다렸다가 +1초 뒤** `finished`를 방출한다. 날아가는 투사체가 없으면 기존 `END_DELAY`(0.6초) 뒤 방출한다. **([투석] 통합 전투는 시간 타이머 대신 순차 연출이 종료를 구동한다 — 아래.)**

### 랑그릿사1(MD)식 연출 — 액자 · HUD · 템포

전투 오버레이는 랑그릿사 1(메가드라이브)의 전투 화면 느낌을 따른다. **전투 판정·능력치 시스템([CombatResolver](combat.md)·[BattleSim](#헤드리스-전투-결산-battle_simgd-순수)·[Human](../entities/Human.md)·무기)은 바꾸지 않고**, 오버레이의 화면 틀·연출·템포만 이 형태로 구성한다. (원작은 상단 전장에서 병사들이 중앙으로 몰려가 난전하고, 하단에 지휘관 정보 액자가 고정돼 있으며, 전투 한 판이 ~5초로 경쾌하다.)

- **화면 액자**: 화면을 **상단 전장(arena, 높이 = 뷰포트 높이 × `ARENA_FRACTION` = 0.62)** + **하단 지휘관 HUD**(나머지)로 나눈다. 모든 전투 토큰(멤버·투석기·구조물)은 **arena 높이 안에서만** 스폰·이동한다(세로 열 y 범위가 `arena_h` 기준으로 압축됨). 화면 밖 돌격 x 오프셋(`CHARGE_OFFSET`)은 그대로 — 근접 유닛은 여전히 화면 밖에서 중앙으로 돌격해 난전한다.
- **분산 난투(다채로운 교전)**: 원작처럼 병사들이 전장 전체에 흩어져 여기저기서 교전하도록 한다.
  - **토큰 크기**(`TOKEN_R`=30) — 랑그릿사 화면 비율에 맞춘 크기. 분산 난투라 커도 한 점에 뭉치지 않는다.
  - **지면 배치 + 2D 산포 스폰** — 유닛은 arena를 하늘/지면으로 나눈(`GROUND_FRACTION`=0.5) **지면 밴드 안에서만** 세로 산포(하늘에 뜨지 않게) + 소량 지터. 돌격 유닛은 index마다 화면 밖 x를 어긋나게(파도치듯 시차 도착), 라인 유닛은 x에 지터. 한 줄로 뭉치지 않는다. 공성 유닛·성벽 구조물도 지면 밴드에 배치.
  - **듀얼 타겟팅**(`_assign_duels`/`_target_for`) — 전투 시작 시 양 팀 멤버를 y 순서로 **1:1 짝(`duel`)** 지어(인원 다르면 순환 배정), 각자 **자기 짝을 향해** 돌격·교전한다. 짝이 죽으면 **최근접 적으로 폴백**. 이래야 근접 유닛이 최근접 적 하나로 우르르 몰리지 않고 전장에 퍼진다.
  - **개인 교전 지점**(`engage_off`) — 접근 시 대상 자체가 아니라 대상 주변의 유닛별 소량 오프셋 지점으로 다가가, 한 점에 겹쳐 쌓이지 않고 둘러싼다.
  - **지면 하한 클램프** — 매 프레임 유닛 `pos.y`를 지면 밴드 하한(`_arena_h − TOKEN_R×1.5`)으로 가둬, 이동·밀림으로도 토큰·HP 바가 하단 HUD를 침범하지 않게 한다.
  - **깊이 정렬(z-index)** — 토큰 `z_index = int(pos.y)`. y가 아래일수록 앞에 그려 2.5D 원근감을 준다(스폰·매 프레임 갱신). 투사체(`Z_PROJECTILE`=3000)·데미지 숫자(`Z_FLOAT`=4000)는 토큰 z(≤ arena 높이)보다 훨씬 위에 둬 유닛에 가리지 않게 한다.
  - **근접 밀림 + 펜싱 리듬** — 근접 피해 타격 시:
    1. 공격자는 **돌진(`_lunge`, 앞으로 갔다 복귀)** 모션을 낸다.
    2. **피격자만** 공격자 반대쪽으로 `HIT_PUSH_PX`(=`TOKEN_R×2` = 병사 폭, ≈60px) 밀린다(공격자는 즉시 안 붙음).
    3. 공격자는 `ADVANCE_DELAY`(0.5초, 실시간) 동안 제자리 정지(`approach_hold`) 후 **다시 접근해 붙는다**.
    - 밀림은 sim pos를 즉시 바꾸되(판정 정확), 토큰은 시각 오프셋(`voff`)을 밀린 만큼 얹어 이전 위치에서 시작해 `PUSH_SLIDE_SPEED`(500px/s)로 0까지 감쇠하며 **부드럽게 슬라이드**한다(일반 이동은 `voff`=0이라 pos 직행 — 랙 없음). 원거리·투척·공성 피격은 위치 이동·홀드 없이 반짝임(`_flash`)만([Combat Feedback](combat-feedback.md)).
  - **트리클 진입**(근접·궁수 공통) — 화면 밖 스폰 x를 index마다 `ENTRY_STAGGER_PX`(170px)씩 크게 벌려, 한꺼번에 몰려오지 않고 **한 명씩 시차를 두고 진입**하게 한다(랑그릿사 느낌).
  - **대열 진입**(궁수·완드 전원 + 원거리 교전의 근접 유닛) — 대상 유닛은 화면 밖에서 진입해 **대열 슬롯**(`_formation_slot`)으로 이동하고, **도착하면 스냅·정지**한다(`formed` 플래그). 궁수는 도착 즉시 사격, 원거리 교전에서 사거리 미달인 근접 유닛은 대형에 선 채 대기(공격 못 하고 사격당함). 대열 진입 이동은 **사거리 게이트보다 먼저** 처리해, 대기 근접 유닛도 대형까지는 이동한다(도착 전엔 사격 안 함). 대열 이동은 **근접 돌격과 동일한 화면 속도**로 한다(raw `delta`에 `MELEE_PLAYBACK`을 곱해 원거리 재생 배속 1.0을 상쇄 — 근접 유닛과 같은 속도로 자리잡음). 슬롯은 **자기 진영 뒤쪽**(좌측 부대 x≈0.27에서 왼쪽, 우측 부대 x≈0.73에서 오른쪽)에 `FORMATION_PER_ROW`(3) 행마다 안쪽으로 들여쓴 **계단식**으로 만들고 **좌측 부대(a)는 좌우 반전**. 행 내 x 간격은 넓게(`TOKEN_R×4.8`). 근접 모드에서 적이 `CHARGE_RANGE_PX` 안에 들면 자리잡은 뒤에도 근접 전환(돌격)한다.
  - **주의(생존 결과 변동)**: 듀얼 타겟팅은 오버레이의 *누가 누굴 공격하나*를 바꾸므로 **오버레이 전투의 생존 결과가 종전과 달라진다**(판정 공식·능력치는 불변, 헤드리스 `BattleSim`도 불변 — NPC끼리 전투엔 영향 없음). 관전 전투는 원래 헤드리스와 다른 근사라 허용.
- **배경**: 검은 dim 대신 **전장 플레이스홀더**(하늘 밴드 + 지면 밴드, 단색). 지형 아트는 `미구현`(플레이스홀더). 아래 월드맵 클릭 흡수(`MOUSE_FILTER_STOP`)는 유지(관전 전용).
- **템포(근접 2배속, 판정 불변)**: 근접 전투는 `_process`에서 **시뮬을 `delta × MELEE_PLAYBACK`(2.0)로 진행**해, `BATTLE_TIME`(시뮬 10초)을 **실시간 ≈5초**에 재생한다. 균일 시간 스케일이라 공격·RNG 소비 순서가 10초 재생과 동일 → **생존 결과 불변**(원거리 2배속과 같은 원리). 원거리는 기존 방식(`RANGED_BATTLE_TIME` 5초 + 쿨다운 ÷ `RANGED_SPEED`)을 그대로 쓰고 `_playback`=1.0. 투사체·lunge·넉백 tween은 실시간이라 배속을 안 타지만(시각만), 판정엔 영향 없다.
- **하단 지휘관 HUD (3분할)** — `[좌 지휘관 패널][중앙 보정 박스][우 지휘관 패널]`. 좌=공격측(a), 우=방어측(b).
  - **지휘관 패널**: 초상화 자리(팀색 사각 플레이스홀더 — 초상화 아트 `미구현`) + 부대명(`party_name`) + `LV`(`commander.level`) + **AT/DF**(`CombatResolver.attack_power/defense` = 버프 반영 실효치) + **큰 병력 수 숫자**(생존 멤버 수) + **상단 병력 바**(생존 ÷ 총원). 병력 수·바는 **매 프레임 실시간 갱신**(멤버 사망 즉시 감소).
  - **중앙 보정 박스**: `MODIFIER_CYCLE`(≈1초)마다 항목을 **순환** 표시한다 — `기본능력`(버프 제외 기본 AT/DF) ↔ `지휘보정`(어느 한쪽이라도 `in_command`면 양측 +델타). 버프가 없으면 `기본능력`만 순환(1칸). 중앙에 "VS".
  - **미구현**: 원작의 `레벨보정`·`지형효과` 박스는 전투 공식에 미반영이라 순환에서 **생략**한다(그 시스템 도입 시 칸 추가). 지휘관 패널 "기본치" AT/DF는 표시용으로 `battle.gd`가 버프 배율(경계·지휘)을 뺀 값을 계산한다 — `CombatResolver`는 **읽기 전용**(수정 안 함). 이 기본치엔 경계(alert) 버프 제외도 포함되므로 `지휘보정` 델타에 경계분이 섞일 수 있다(느낌 표시라 허용).
- **사망 시체 잔류**: `_kill`은 노드를 지우지 않고 알파 `DEATH_ALPHA`로 남긴다(기존) — 여기에 **살짝 눕는 회전**을 더해 MD1식 "바닥에 쓰러진 시체" 느낌을 보강한다.

### 투석 오버레이 순차 연출

`include_siege=true`로 시작한 오버레이 전투(내부 플래그 `_siege_battle`)(투석기 vs 투석기 결투·성벽 폭격·성문 파쇄)는 **프레임 시뮬(`_process`)·전투 타이머를 쓰지 않고** 스크립트 시퀀스로 진행한다. 이 거리(투석기 밴드 4~5·충차 밴드 1)에서는 사람(`members`)은 사거리 게이트·표적 부재로 어차피 행동하지 않으므로 **공성 유닛·구조물만** 움직인다. → [siege-engines.md](siege-engines.md#battlegd-통합-전투--투석기구조물-전투원)

1. **공격자(a) 선공** — `BattleField.firing_siege(_units, "a", _distance)`(밴드 안 살아있는 공성 전투원)이 일제히 1발씩. 발사 판정은 `_ranged_mode`가 아니라 **밴드 소속**(`min_range ≤ _distance ≤ fire_range`)이라 투석기(4~5)·충차(1) 모두 발사한다.
2. **착탄 = 타격 판정** — 투석 볼리는 피해를 **발사 즉시가 아니라 투사체 착탄 시점**에 적용한다(느린 투사체와 hp 감소가 어긋나지 않도록). 착탄 시 표적 hp 차감·데미지 숫자·피격 반응·사망(넉백). 이미 죽은 표적에 도달한 투사체는 무시.
3. **방어자(b) 반격** — 공격자 볼리가 **전부 착탄한 뒤** `SIEGE_COUNTER_GAP`(0.5초) 간격을 두고, **살아남은** 방어측 밴드 공성 전투원이 반격 1발. 성벽·성문(`structure`)은 반격하지 않으므로, 방어측에 밴드 안 공성 전투원이 없으면 반격을 생략한다.
4. **선제 이점** — 공격자→방어자 순차이므로, 공격자 볼리에 방어측 공성 전투원이 전멸하면 반격이 없다(공격자 우선). *(헤드리스 [BattleSim](#헤드리스-전투-결산-battle_simgd-순수) 볼리는 여전히 동시·상호 반격 — 플레이어 비관전 근사.)*
5. **종료** — 마지막 볼리(반격, 없으면 선공)가 착탄한 뒤 `ARROW_LAND_DELAY`(1초) 후 생존자를 반영하고 `finished`를 방출한다.
- 토큰은 팀 색 사각 + 현재 hp 라벨. **원거리(활·완드)는 제자리 사격(단, 근접 모드에서 적이 `CHARGE_RANGE_PX` 안에 들면 근접 전환·돌격), 투척 무기는 접근 중 투척 후 근접 전환, 그 외 근접은 리치 사거리까지 돌격**. LOS는 미구현.
- **타격 연출**(대미지 숫자·반짝임·흔들림·돌진·상태이상 텍스트·사망 넉백)은 [Combat Feedback](combat-feedback.md) 참조. 연출은 판정·생존 결과에 영향을 주지 않는다.
- **생존자 생명점은 전투 후 지속**된다 — 종료 시 각 생존 유닛의 최종 hp를 `Human.hit_points`에 되쓴다(`maxi(1, hp)`). 다음 전투는 이 값에서 시작(`_make_unit`이 `hit_points`를 읽음). 사망자는 `members`에서 빠진다. 회복·자연 재생은 `미구현`([Human](../entities/Human.md) `max_hp()`가 향후 상한 기준).

## 전투 실행·결과 반영 (`game.gd`)

- `_run_battle(attacker, defender)`(awaitable) — 오버레이를 띄우고(`_in_battle` 잠금) `finished`를 `await`한 뒤 생존자를 반영하고 오버레이를 닫는다. 플레이어 개시 전투는 이 함수를 쓰고, 공격 페이즈에서 플레이어가 낀 전투도 순차로 `await`한다.
- `_apply_survivors(party, survivors)` — 각 부대의 `members`를 **생존자로 교체**한다. 지휘관이 사망했으면 생존 멤버 중 첫 명으로 재지정한다. 교체 후 `queue_redraw()`로 토큰을 다시 그린다 — **맵 인원수 배지**(`Party.shows_member_count`)가 사상을 반영하도록(누락 시 초기값에 고착).
  - **NPC 부대가 전멸**하면 맵에서 제거한다(토큰 `queue_free`, `_npc_parties`에서 제외).
  - **플레이어 부대가 전멸**하면 NPC와 마찬가지로 맵에서 제거한다(`_units`에서 제외, `queue_free`). 그 부대가 **활성 부대(`party`)** 였으면 선택을 해제하고 `party`를 **남은 살아있는 플레이어 부대**로 재할당한다(없으면 `null`, [부대 정보](party-info.md) 닫기). **부대가 0이 돼도 패배가 아니다** — 다른 거점 방어 부대를 [분할](party-composition.md)해 다시 움직일 수 있고, 패배는 [세력 소멸](victory.md)(거점 0)로만 난다.
    - `party == null`을 허용한다 — 활성 부대를 잃으면 `party`가 null이 되며, 이때 항상 선택도 해제(`_selected = false`)해 선택 게이트 뒤의 `party` 사용이 안전하다. 무조건 `party`를 참조하던 `_handle_click`은 null 가드를 둔다. 부대를 클릭하면(멤버 있는 부대) `party`가 그 부대로 재할당된다([Selection & Movement](selection-and-movement.md) FOCUS_PARTY).
- **점령(근접 승리)**: `[공격]`(근접)으로 개시해 **수비 NPC가 전멸하고 공격 부대가 생존**하면, 공격 부대를 **수비가 있던 타일로 이동**시킨다(전투는 수비 타일에서 벌어진 것으로 간주). **사격 승리는 이동 없음**(제자리).
- **약탈(노획)**: 전투로 **한 부대만 전멸**하면 승자가 패자 화물·전사자 장비를 노획한다([Raid](raid.md)) — `_apply_survivors`가 패자를 제거(`queue_free`)하기 **전에** 처리한다(전사자 장비를 읽어야 하므로). 플레이어 승자는 선택 패널, NPC 승자는 자동 전량.
- 전투 후 안개·부대 일람을 갱신한다.

## BattleField 헬퍼 (`battle_field.gd`, 순수)

전투씬의 공간 판정을 노드 없이 계산하는 순수 함수(테스트 용이). 유닛은 `{team, alive, pos, human}` 형태의 Dictionary로 다룬다.

- `nearest_enemy(unit, units) -> Dictionary` — `unit`과 **다른 팀**의 **살아있는** 유닛 중 `pos` 거리가 가장 가까운 것. 없으면 빈 Dictionary(`{}`). 오버레이가 매 프레임 대상 재탐색(상대 사망 시 새 상대)에 쓴다. *(공성 전투원(`siege`)·성벽 구조물(`structure`)은 표적에서 제외 — 이들은 [투석 볼리](siege-engines.md#battlegd-통합-전투--투석기구조물-전투원)만 노린다.)*
- `bombard_targets(unit, units, n) -> Array` — [투석기 광역 공격](siege-engines.md#battlegd-통합-전투--투석기구조물-전투원) 표적(한 발 최대 `n`명). 다른 팀 살아있는 유닛 중 **적 공성 전투원(`siege`)을 우선**, 그다음 일반 유닛·**성벽 구조물(`structure`)**, 각 그룹 내 거리순으로 최대 n명(없으면 빈 배열). `nearest_enemy`와 달리 **공성 전투원·구조물을 표적에 포함**한다(투석기가 적 투석기·성벽을 노림).
- `team_wiped(units, team) -> bool` — 그 팀에 살아있는 (비공성) 유닛이 하나도 없으면 `true`. **성벽 구조물은 셈에 포함**(성벽만 있는 방어 팀은 성벽 파괴 시 전멸), **공성 전투원(`siege`)은 제외**(승패는 Human/구조물 기준).
- `survivors(units, team) -> Array` — 그 팀의 살아있는 유닛들의 `human` 목록. 공성 전투원·구조물(`human` 없음)은 제외.
- `archer_should_charge(unit_range, dist, threshold) -> bool` — 사거리 ≥ 2인 유닛이 최근접 적과의 거리 `dist`가 `threshold` 이하이면 `true`(근접 전환 판정). 사거리 < 2(이미 근접)면 항상 `false`.
- `firing_siege(units, team, distance) -> Array` — 그 `team`의 **살아있는 공성 전투원(`siege`) 중 밴드 안**(`min_range ≤ distance ≤ fire_range`) 유닛 목록. 오버레이 [투석 순차 연출](#투석-오버레이-순차-연출)이 한 진영의 발사 대상·반격 가능 여부(빈 배열이면 반격 없음) 판정에 쓴다. 구조물(`structure`)·일반 유닛은 제외(`siege` 플래그만).

## 미구현

- 지휘범위·지형·마법 위력 공식 등 [combat.md](combat.md)의 미구현 항목. 치명타 연동 상태이상(출혈·기절)은 [status-effects.md](status-effects.md)로 도입됨(전투씬 내 완결, 월드맵 이월은 미구현).
- **헤드리스 결산은 근사** — 위치·이동을 무시하므로 오버레이 결과와 다를 수 있다(플레이어가 안 보는 NPC끼리 전투에만 적용).
- 전투 애니메이션 디테일(타격 이펙트 등). 승패는 [세력 소멸](victory.md)(거점 0)로만 난다 — 부대 전멸은 게임오버가 아니다(위 `_apply_survivors`).

- [정상] 압도적인 A(항상 명중·일격)면 10초 내 B 전멸, A 생존자 존재

- [정상] 근접 교전(`distance=1`, 기본) — 사거리와 무관하게 전원 공격(근접·원거리 모두)
- [정상] 생존자 목록은 원래 멤버의 부분집합
- [경계] 양측 전원 회피(항상 빗나감)면 10초간 공격해도 **아무도 안 죽는다**(양측 전원 생존)
- [경계] 한쪽 멤버가 비어 있으면 다른 쪽 전원 생존
- [정상] 같은 시드 → 같은 결과(결정적)
- [정상] 출혈 도트 — 참격 치명 확정 셋업에서 도트가 더해져 방어측이 더 빨리 전멸한다 ([status-effects.md](status-effects.md))
- [정상] 기절 — 타격 치명으로 기절한 유닛은 기절 지속 동안 공격을 걸지 못한다
- [정상] hp 지속 — 피해를 입고 **생존한** 유닛의 `Human.hit_points`가 전투 후 감소해 있고(풀피 아님) `1 ≤ hit_points ≤ max_hp()`
- [정상] hp 지속 — 피해를 전혀 안 받은 생존자는 `hit_points` 불변
- [정상] **투석 표적 선정(순수, 5g-B)** — `bombard_pick(적 투석기, 적 멤버, 5)`은 적 투석기를 먼저(대포병 우선) 그다음 멤버, 최대 5개; 적이 5 초과면 앞쪽 5개, 없으면 빈 배열
- [경계] **밴드 밖 미발사** — 거리 1(밴드 4~5 밖)에서 투석기끼리 붙으면 볼리 없음 → 양측 `SiegeUnit.hit_points` 불변
- [정상] **밴드 안 명중(시드)** — 거리 4에서 명중 시드로 볼리 시 적 투석기(우선 표적) `hit_points`가 `rolled_damage`만큼 감소
- [정상] **상호 반격** — 거리 4에서 양측 투석기 1대씩·명중 시드면 양쪽 다 `hit_points` 감소(한쪽만 깎이지 않음)
- [경계] **siege 없음 회귀** — `a_siege`·`b_siege` 생략 시 결과가 기존 멤버 전투와 동일(볼리 프리페이즈 스킵)

### 전투 흐름 (실행 확인)
- 오버레이를 헤드리스로 돌려 **한 팀이 전멸하거나 10초(`BATTLE_TIME`)에 종료**되고 생존자 집계가 타당한지 확인(양측 전원 회피면 10초 꽉 채우고 전원 생존).
- **원거리 유닛은 이동하지 않고 제자리에서 반복 사격**한다 — 최종 위치가 시작 위치와 같고 쿨다운마다 피해가 들어가는지 확인.
- **리치 긴 무기가 선제**한다 — 장창(리치 2.0) vs 검(리치 1.2) 접근 시 장창이 먼저 피해를 주는지 확인.
- **투척 무기 유닛은 접근 중 최대 3회 투척 후 근접 리치 도달 시 주무기 근접**으로 전환하는지 확인.
- **궁수 근접 전환**: 근접 모드에서 순수 궁수가 적이 `CHARGE_RANGE_PX` 안에 들면 제자리 사격을 멈추고 접근·근접 공격으로 바뀌는지 확인.
- **원거리 2배속**: 원거리 전투가 5초에 종료되고, 근접 10초 전투와 발사·처치 양상이 비슷한지 확인.
- **화살 속도·종료 대기**: 화살이 기존의 절반 속도로 날고, 종료 시점에 날아가던 화살이 착탄한 뒤 약 1초 후 전투가 닫히는지 확인.
- **개별 병사 렌더**: 팀 멤버가 각자 토큰으로 세로 열에 서고(각 토큰 = hp 숫자 + HP 바), 접근·교전 시 각자 움직이며, 죽으면 넉백·페이드하는지 확인. 기절=회색·출혈=붉은 tint. 투석기·성벽 토큰도 개별 표시.
- **오버레이 스모크**(`test_battle_overlay.gd`): `start(부대A, 부대B)` 후 `_process`를 수동 펌프하면 크래시 없이 한 팀이 전멸해 `_running`이 거짓이 되고 생존자 집계가 타당한지 확인. **팀 멤버 수만큼 렌더 노드가 생기는지**(팀당 1개 묶음이 아니라) 검증.
- **화면 밖 돌격**(`test_battle_overlay.gd`): 근접 교전에서 근접 공격자(팀 a)가 화면 왼쪽 밖(`pos.x < 0`)에서 스폰하고, 각 멤버 `speed`가 `UNIT_SPEED`의 ±20% 범위이며, 펌프 시 크래시 없이 종료되는지 확인.
- **궁수 대열 진입**(`test_battle_overlay.gd`): 원거리 교전에서 궁수가 화면 밖(`pos.x < 0`, 팀 a)에서 진입해 아직 미형성(`formed=false`)이고, 펌프하면 대열 슬롯에 스냅(`formed=true`, `pos.x==formation.x`)한 뒤 더 펌프해도 `pos`가 변하지 않는지(자리잡고 제자리 사격) 확인.
- **MD1 액자·HUD·템포**(`test_battle_overlay.gd`):
  - [정상] `start` 후 모든 멤버 토큰이 arena 높이 안(`pos.y ≤ arena_h`)에 스폰된다.
  - [정상] HUD 생성 — 좌/우 패널 병력 수(`_hud_count("a")`)가 초기 멤버 수와 같다.
  - [정상] 멤버 사망 후 그 팀 병력 수가 감소한다(`_hud_count`가 초기값보다 작아짐).
  - [정상] 근접 교전 `_playback == MELEE_PLAYBACK`, 원거리 교전 `_playback == 1.0`.
  - [정상] 근접 `_process(delta)` 1회가 `_elapsed`를 `delta × MELEE_PLAYBACK`만큼 진행(2배속 재생).
  - [정상] 패널 AT/DF(`_panel_at/_panel_df`)가 지휘관 실효치(`CombatResolver.attack_power/defense`)와 일치.
  - [정상] 중앙 보정 박스 라벨: 버프 없으면 `["기본능력"]`, 한쪽이라도 `in_command`면 `["기본능력","지휘보정"]`.
  - [정상] 각 멤버에 `duel`(상대 팀 유닛)이 배정된다; 짝이 죽으면 `_target_for`가 최근접 적으로 폴백.
  - [정상] 산포 스폰 — 같은 팀 근접 유닛들의 y가 서로 다르다(한 줄로 겹치지 않음).
- 개시(인접 적 클릭)→전투→사상자 반영→복귀, NPC 공격 페이즈, NPC끼리 헤드리스 즉시 결산은 `game.gd` 배선이라 실제 실행으로 확인한다.
- **플레이어 전멸 처리**(`_apply_survivors`): 전멸한 플레이어 부대가 `_units`에서 사라지고, 활성 부대였으면 남은 부대로 전환(없으면 `party==null`이나 클릭·크래시 없음), 부대 0이어도 게임 계속(캠프에서 재편 후 복귀)은 `game.gd` 배선이라 실제 실행으로 확인한다.
